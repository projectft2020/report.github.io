# 記憶系統

## 核心概念

- **三層架構**: MEMORY.md（主索引）→ topics/（結構化知識）→ daily/（完整日誌）
- **時間 + 主題雙軸**: 時間軸追溯 + 主題深度學習
- **QMD 向量搜索**: 語義搜索，快速定位相關記憶

## 關鍵決策

### 2026-03-17: 記憶系統重構 - 方案 D 改進版

**背景**:
- GPU OOM 錯誤頻繁（kanban-workspace 索引 563 個文件）
- 記憶系統缺乏結構化知識
- 需要建立長期可維護的記憶系統

**理由**:
- 重建 QMD collections，只索引最近 7-14 天（memory-daily）
- 建立 topics/ 目錄，結構化知識庫
- 簡化架構，減少維護負擔

**結果**:
- 移除 kanban-workspace collection（563 files → 0）
- 清理空 collections（27 個）
- 建立 topics/ 目錄結構
- GPU OOM 問題解決

## 學習點

- **模式提取**: 記憶系統需要「快速搜索 + 結構化知識 + 完整追溯」三個層級
- **QMD 最佳實踐**:
  - 避免索引過多文件（< 100 files）
  - 分類 collections（daily/topics/archive）
  - 定期清理空 collections

## 系統架構

```
memory/
├── daily/              # 完整日誌（不壓縮）
├── topics/             # 結構化知識（每天提取）
├── archive/            # 歸檔（> 30 天）
└── MEMORY.md           # 主索引
```

## QMD Collections

- **memory-daily**: 最近 7-14 天的 daily/ 文件
- **memory-topics**: topics/ 所有文件（永久索引）
- **memory-archive**: archive/ 文件（按需索引）
