- **任務名稱：** 研究知識庫綜合分析 - 每週更新
- **執行頻率：** 每 7 天（604,800,000 毫秒）
- **下次執行：** 2026-03-11（7 天後）
- **通知設置：** ✅ 已啟用（完成後自動匯報）

#### 2.2 Cron 任務配置

```json
{
  "name": "研究知識庫綜合分析 - 每週更新",
  "schedule": {
    "kind": "every",
    "everyMs": 604800000  // 7 天
  },
  "sessionTarget": "isolated",
  "payload": {
    "kind": "agentTurn",
    "message": "執行研究知識庫綜合分析：\n\n1. 運行 python3 kanban-ops/research_knowledge_base.py\n2. 運行 python3 kanban-ops/quant_applicability_identifier.py\n3. 分析新完成的研究報告\n4. 跟蹤研究趨勢變化\n5. 識別新的量化交易應用機會\n\n輸出兩個報告：\n- research-knowledge-base-summary-YYYYMMDD.md\n- quant-applicability-report-YYYYMMDD.md\n\n完成後向用戶匯報關鍵發現和變化趨勢。",
    "model": "zai/glm-4.7",
    "thinking": "medium"
  },
  "delivery": {
    "mode": "announce"
  },
  "notify": true
}
```

#### 2.3 定期更新流程

**每週自動執行的任務：**

1. **運行知識庫綜合分析**
   - 執行 `python3 kanban-ops/research_knowledge_base.py`
   - 掃描所有已完成的研究報告
   - 提取元數據和內容分析
   - 應用四維度評分系統
   - 生成綜合性知識報告

2. **運行量化應用潛力識別**
   - 執行 `python3 kanban-ops/quant_applicability_identifier.py`
   - 識別量化交易應用潛力
   - 評估應用場景和實現難度
   - 更新應用案例庫

3. **生成更新報告**
   - 研究知識庫綜合分析報告
   - 量化交易應用潛力評估報告
   - 趨勢變化分析

4. **自動匯報**
   - 向用戶發送關鍵發現
   - 提供變化趨勢分析
   - 建議下一步行動

#### 2.4 輸出文件格式

每次更新會生成兩個報告文件（帶日期戳記）：

1. **研究知識庫綜合分析報告**
   - 文件：`research-knowledge-base-summary-YYYYMMDD.md`
   - JSON：`research-knowledge-base-summary-YYYYMMDD.json`
   - 內容：總體統計、Top 15 高分論文、關鍵詞統計、按年份分布、應用建議

2. **量化交易應用潛力評估報告**
   - 文件：`quant-applicability-report-YYYYMMDD.md`
   - JSON：`quant-application-library-YYYYMMDD.json`
   - 內容：應用潛力統計、按領域統計、高優先級應用、實施建議、技術路線圖
