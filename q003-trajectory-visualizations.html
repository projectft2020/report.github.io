概述¶

本文件展示了如何使用 memory/vector_search.py 腳本來視覺化研究主題在時間序列上的演變過程。通過語義搜索找到相關記憶，並生成可視化軌跡，幫助理解研究主題的發展歷程。

🔧 腳本使用方法¶

基本概念¶

VectorTrajectory 類是一個用於追蹤研究主題演變過程的工具。它能夠：

  • 掃描記憶文件（Markdown 格式，以日期開頭的文件）
  • 基於關鍵詞匹配找到相關文檔
  • 生成時間序列的軌跡可視化
  • 導出結果為 Markdown 或 JSON 格式

初始化¶

from memory.vector_search import VectorTrajectory

# 創建實例（可指定記憶路徑）
vt = VectorTrajectory(memory_path="/Users/charlie/.openclaw/workspace/memory")

主要功能¶

1. 查找軌跡 (find_trajectory)¶

# 查找特定主題的軌跡
trajectory = vt.find_trajectory("Druckenmiller 宏觀趨勢識別系統", max_results=15)

# 返回結果格式：按時間排序的軌跡點列表
# 每個軌跡點包含：id, date, title, content, summary, confidence

2. 生成 ASCII 可視化 (visualize)¶

# 生成 ASCII 格式的可視化
visualization = vt.visualize("統計套利策略")
print(visualization)

3. 導出結果 (export)¶

# 導出為 Markdown 格式
markdown_path = vt.export("美林時鐘模型", format="md")

# 導出為 JSON 格式  
json_path = vt.export("Paul Tudor Jones 1987 Black Monday 預測模型", format="json")

完整範例¶

#!/usr/bin/env python3

from memory.vector_search import VectorTrajectory

def analyze_topic_trajectory(topic):
    vt = VectorTrajectory()

    # 查找軌跡
    trajectory = vt.find_trajectory(topic)

    if trajectory:
        # 顯示可視化
        print(vt.visualize(topic))

        # 導出報告
        output_path = vt.export(topic, format="md")
        print(f"報告已保存至: {output_path}")
    else:
        print(f"找不到主題 '{topic}' 的相關軌跡")

# 使用範例
analyze_topic_trajectory("Druckenmiller 宏觀趨勢識別系統")

🔍 如何解讀軌跡圖¶

軌跡圖結構解析¶

1. 時間軸（橫軸）¶

  • 軌跡按時間順序排列，從最早到最新
  • 每個點代表一個相關的記憶文檔
  • 日期格式：YYYY-MM-DD

2. 相關性強度（縱軸）¶

  • 置信度分數表示查詢與文檔的相關性
  • 分數範圍：0.00（無相關）到 1.00（高度相關）
  • 目前版本使用簡化的關鍵詞匹配算法

3. 內容摘要¶

  • 每個軌跡點包含文檔的摘要信息
  • 摘要從文件第一行有效內容生成
  • 提供快速了解文檔主題的途徑

軌跡模式分析¶

單點軌跡¶

✓ 2026-02-20 (0.75): 研究主題挖掘 - 2026-02-20
  • 意義: 主題只在單一時間點被記錄
  • 解讀: 可能是新興主題或單次研究記錄

多點軌跡¶

→ 2026-02-19 (0.60): 初步概念討論
→ 2026-02-20 (0.85): 詳細方案制定
✓ 2026-02-21 (0.90): 實施與優化
  • 意義: 主題持續發展，呈現進化過程
  • 解讀: 研究深度和複雜性隨時間增加

高置信度軌跡¶

✓ 2026-02-20 (0.95): 核心策略實現
  • 意義: 文檔與查詢高度相關
  • 解讀: 找到了精確匹配的核心內容

低置信度軌跡¶

→ 2026-02-20 (0.20): 相關背景資料
  • 意義: 文檔與查詢弱相關
  • 解讀: 可能是相關但非核心的背景資訊
