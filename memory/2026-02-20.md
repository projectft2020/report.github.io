## Kanban 歸檔系統部署（02:40）

### 問題識別
- tasks.json 檔案持續增長（目前 79 個任務，77.10 KB）
- 如果每天完成 30+ 任務，一個月後會超過 1MB
- 影響查詢性能和系統效率

### 解決方案
創建了完整的 Kanban 歸檔系統：

**歸檔策略：**
- 保留：最近 7 天的已完成任務 + 所有 In Progress + 所有 Pending
- 月度歸檔：7-30 天前的已完成任務（archive/tasks-YYYY-MM.json）
- 壓縮歸檔：30 天前的已完成任務（archive/tasks-compressed-*.json.gz）

**自動觸發：**
- 文件大小超過 500 KB 時自動執行歸檔
- 或每月定期執行（Cron/Heartbeat）

**空間節省：** 90%+

### 已創建文件
1. **archive_tasks.py** - 歸檔腳本（v2.0）
   - 支援 --stats, --dry-run, --force 模式

### 策略調整：並行模式 → 快速循環模式

#### 用戶反饋
- **核心擔憂**：多 agent 並行太冒險，可能因為使用過度被 ban
- **新思路**：不增加同時並行的數量，而是減少空白等待的時間

#### 舊策略（已放棄）
- 並行觸發：一次觸發 3-5 個 agent
- 風險：Rate limit、被 ban
- 預期完成：10-20 個任務（風險高）

#### 新策略（推薦）
- 快速循環：每 10 分鐘檢查一次
- 並行上限：1-2 個 agent
- 預期完成：36-72 個任務（穩定）

#### 優勢
- ✅ 保持低並行（1-2 個）
- ✅ 縮短空白等待時間
- ✅ 充分利用 6 小時
- ✅ 更安全，不會被 ban

#### 實施方案

**修改 turbo_mode.py：**
```python
def run_loop_mode(self, duration_hours=6):
    """快速循環模式"""
    check_interval = 10  # 每 10 分鐘
    max_concurrent = 1    # 最多 1-2 個並行
    
    for i in range(duration_hours * 60 // check_interval):
        # 檢查待觸發任務
        ready_tasks = self.get_ready_tasks(limit=2)
        
        # 觸發任務
        for task in ready_tasks:
            self.spawn_task(task)
        
        # 等待
        time.sleep(check_interval * 60)
```

**修改 TURBO_TASKS.json：**

```json
    "max_concurrent": 1,          // 最多 1-2 個並行
    "max_tasks_per_check": 2     // 每次最多觸發 2 個
  }
}
```

### Scout Agent 協調問題

#### 問題識別

**現有機制（事件驅動監控）：**
```python

if pending_tasks < 3 and time_since_last_scan > 2 hours:
    trigger_scout_scan()
```

**潛在問題：**

**核心問題：**
- Scout 的 2 小時保護會導致任務耗盡
- 加速模式會沒有任務可執行
- 浪費大量時間

#### 解決方案

**方案 1：調整 Scout 觸發條件**
```python
# monitor_and_refill.py 改為

def should_scan(self, pending_count, last_scan_time):
    # 檢查是否在加速模式
    turbo_mode = self.is_turbo_mode_active()
    
    if turbo_mode:
        # 加速模式：更快補充
        threshold = 5  # 待辦 < 5 個就掃描
        min_interval = 30 * 60  # 最少 30 分鐘
    else:
        # 正常模式：保守掃描
        threshold = 3  # 待辦 < 3 個才掃描
        min_interval = 2 * 60 * 60  # 最少 2 小時
    
    time_since_scan = datetime.now() - last_scan_time
    
    return (pending_count < threshold and 
            time_since_scan > timedelta(seconds=min_interval))
```

**方案 2：Scout 獨立預熱**
```python
def start_turbo_mode(self):
    # 1. 先觸發 Scout 掃描（預熱）
    self.trigger_scout_scan(force=True)
    self.wait_for_scout_completion()  # 等待完成
    
    # 2. 開始循環模式
    self.run_loop_mode()
```

**方案 3：混合方案（推薦）**
- 啟動時：強制 Scout 掃描（預熱）
- 循環中：待辦 < 5 個時，允許 Scout 掃描（最小間隔 30 分鐘）
- 安全限制：每小時最多 2 次 Scout 掃描，看板最多 20 個 pending 任務
