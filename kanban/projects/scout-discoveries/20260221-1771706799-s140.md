## Research Summary

genriesz is a comprehensive Python package for automatic debiased machine learning (DML) that implements Generalized Riesz Regression (GRR) under Bregman divergences. It provides a unified framework for estimating Riesz representers with automatic regressor balancing (ARB) and supports multiple estimators with valid statistical inference.

## Detailed Analysis

### Technical Foundation

genriesz is built on the mathematical foundation of Riesz representation theorem and debiased machine learning. The core idea is to estimate linear functionals of the form θ₀ = 𝔼[m(W, γ₀)] where W = (X, Y) contains regressors and outcomes, and γ₀(x) = 𝔼[Y|X=x] is the unknown regression function.

The package implements Generalized Riesz Regression which estimates Riesz representers α₀ by minimizing empirical Bregman-Riesz objectives. This approach unifies various existing methods including:

- **SQ-Riesz regression**: Using squared distance generators
- **UKL-Riesz regression**: Using unnormalized KL divergence generators  
- **BKL-Riesz regression**: Using binary KL divergence generators
- **BP-Riesz regression**: Using Basu's power divergence generators
- **PU-Riesz regression**: Using PU learning generators

### Key Features

#### 1. Modular Architecture
The package provides a modular interface allowing users to specify:
- **Target linear functional** via a black-box evaluation oracle
- **Representer model** via basis functions (polynomial, RKHS approximations, random forest leaf encodings, neural embeddings, nearest-neighbor catchment basis)
- **Bregman generator** with optional user-supplied derivatives

#### 2. Built-in Estimands
Convenience wrappers are provided for common causal inference problems:
- **ATE** (Average Treatment Effect): `grr_ate`
- **ATT** (Average Treatment Effect on Treated): `grr_att`  
- **DID** (Difference-in-Differences): `grr_did`
- **AME** (Average Marginal Effect): `grr_ame`

#### 3. Basis Functions
Multiple basis types are supported:
- **Polynomial basis**: For polynomial feature interactions
- **RKHS-style bases**: Random Fourier features and Nyström approximations
- **kNN catchment basis**: For nearest-neighbor matching
- **Random forest leaves**: Via scikit-learn integration
- **Neural network features**: Via PyTorch integration

#### 4. Statistical Inference
The package provides valid inference through:
- Cross-fitting support for high-capacity settings
- Wald-type standard errors from empirical influence function scores
- Confidence intervals and p-values for all estimators
- Asymptotic normality guarantees under standard conditions

### Practical Usage Example

```python
import numpy as np
from genriesz import (
    grr_ate,
    UKLGenerator,
    PolynomialBasis,
    TreatmentInteractionBasis,
)

# Example data: X = [D, Z] where D is treatment, Z are covariates
n, d_z = 1000, 5
rng = np.random.default_rng(0)
Z = rng.normal(size=(n, d_z))
D = (rng.normal(size=n) > 0).astype(float)
Y = 2.0 * D + Z[:, 0] + rng.normal(size=n)
X = np.column_stack([D, Z])

# Create basis and generator
psi = PolynomialBasis(degree=2)
phi = TreatmentInteractionBasis(base_basis=psi)
gen = UKLGenerator(C=1.0, branch_fn=lambda x: int(x[0] == 1.0)).as_generator()

# Estimate ATE
res = grr_ate(
    X=X, Y=Y,
    basis=phi,
    generator=gen,
    cross_fit=True, folds=5,
    riesz_penalty="l2", riesz_lam=1e-3,
    estimators=("ra", "rw", "arw", "tmle"),
)
print(res.summary_text())
```

### Comparison to Related Software

genriesz complements existing DML libraries like DoubleML and EconML by:
- **Estimand-centric approach**: Users define the functional directly, making it easy to prototype new targets
- **Explicit representer fitting**: Makes the connection between Riesz estimation and balancing weights transparent
- **Unified framework**: Combines methods from density ratio estimation, covariate balancing, and debiased ML

Unlike libraries focused on heterogeneous treatment effects, genriesz specializes in low-dimensional linear functionals with valid inference via orthogonalization.
