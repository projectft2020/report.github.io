# 自动研究推进规则

## 核心原则
- **主动推进**: 不等待用户指示，自动按队列研究
- **持续进行**: 研究完成立即找下一个主题，不闲下来
- **可行性排序**: 优先研究可行性高、时间短的主题

## 执行流程
1. 研究当前队列中的主题
2. 完成后自动搜索新的 arXiv 论文/研究主题
3. 评估可行性（高/中高/中/低）
4. 按可行性排序加入队列
5. 继续下一个研究

## 队列管理
- 每次保留 3-5 个待研究主题
- 按可行性排序
- 记录在 MEMORY.md 和 kanban tasks.json

## 用户交互
- 研究进度汇报：每日 memory 更新
- 重大发现：立即汇报
- 不再问用户选择研究方向

## 研究优先级
1. **高可行性 + 短时间** (2-3周)
2. **中高可行性 + 中等时间** (3-4周)
3. **中高可行性 + 长时间** (4-5周)
