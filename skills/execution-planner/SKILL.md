# Execution Planner - 執行計畫

> "An agent without a plan drifts" — OPT-1 參考 learn-claude-code s03 TodoWrite

## 核心原則

**先規劃後執行**：在開始任何複雜任務前，先生成明確的執行步驟。

## 使用時機

### 需要執行計畫的任務類型

✅ **需要規劃的任務**：
- 多步驟開發任務（例如：Dashboard 開發）
- 串聯多個代理的任務（例如：Research → Analyst → Creative）
- 跨系統的改動（例如：修改看板系統）
- 重構或架構調整

❌ **不需要規劃的任務**：
- 單一簡單操作（例如：讀取檔案、執行命令）
- 心跳檢查（例行的維護腳本）
- 快速查詢（例如：搜尋記憶）

## 計畫模板

### 模板 1：開發任務計畫

```
## 執行計畫：[任務標題]

### 階段 1: 準備
1. [ ] 讀取相關檔案（列出檔案路徑）
2. [ ] 理解現狀（列出關鍵發現）
3. [ ] 識別改動點（列出需要修改的檔案和函數）

### 階段 2: 設計
1. [ ] 設計解決方案（列出核心邏輯）
2. [ ] 評估風險（列出潛在問題）
3. [ ] 準備測試策略（列出測試方法）

### 階段 3: 實現
1. [ ] 實現步驟 1（具體操作）
2. [ ] 實現步驟 2（具體操作）
3. [ ] ...

### 階段 4: 驗證
1. [ ] 執行測試
2. [ ] 檢查輸出
3. [ ] 確認無副作用
```

### 模板 2：多代理協作計畫

```
## 執行計畫：[任務標題]

### 工作流
[階段 1] → [代理 A] → 輸出路徑
[階段 2] → [代理 B] (使用 A 的輸出) → 輸出路徑
[階段 3] → [代理 C] (使用 B 的輸出) → 最終輸出

### 依賴檢查
- 階段 2 依賴：階段 1 完成
- 階段 3 依賴：階段 2 完成

### 同步策略
- 等待所有前置代理完成
- 集成所有輸出
- 生成最終報告
```

### 模板 3：系統改動計畫

```
## 執行計畫：[改動標題]

### 影響範圍
- 涉及檔案：[列出檔案路徑]
- 依賴模組：[列出模組]
- 潛在影響：[列出其他系統]

### 回滾計畫
- 改動前備份：[備份路徑]
- 回滾步驟：[回滾命令]
- 驗證回滾：[驗證方法]

### 實施步驟
1. [ ] 備份現狀
2. [ ] 修改檔案 1
3. [ ] 修改檔案 2
4. [ ] 測試改動
5. [ ] 確認無副作用
```

## 使用流程

### 在複雜任務開始前

1. **識別任務類型**：判斷是否需要執行計畫
2. **選擇模板**：根據任務類型選擇合適的模板
3. **生成計畫**：填充模板，明確每個步驟
4. **確認計畫**：檢查計畫是否完整和可行
5. **執行計畫**：按照計畫逐步執行
6. **追蹤進度**：標記完成的步驟

### 執行中追蹤

- 完成每個步驟後打勾
- 記錄實際時間 vs 預估時間
- 標記遇到的問題
- 記錄調整的計畫

## 範例

### Dashboard 開發計畫

```
## 執行計畫：Market Score V3 開發

### 階段 1: 準備
1. [x] 讀取現有 Market Score 代碼
2. [x] 理解 V2 架構和限制
3. [x] 識別需要改進的功能

### 階段 2: 設計
1. [ ] 設計 3-tier 文檔結構
2. [ ] 設計緩存機制（_price_cache, _ma_cache, _score_cache）
3. [ ] 準備測試用例（28 個測試場景）

### 階段 3: 實現
1. [ ] 實現核心計算邏輯
2. [ ] 添加類型提示
3. [ ] 實現優雅錯誤處理

### 階段 4: 驗證
1. [ ] 執行所有測試
2. [ ] 檢查測試覆蓋率 > 80%
3. [ ] 確認與現有系統兼容
```

## 常見陷阱

### ❌ 避免這些錯誤

1. **跳過規劃**：直接開始編碼，導致返工
2. **規劃過細**：花費太多時間在細節上
3. **不追蹤進度**：忘記哪些步驟已完成
4. **不靈活調整**：遇到問題不調整計畫

### ✅ 最佳實踐

1. **快速規劃**：5-10 分鐘生成計畫
2. **明確步驟**：每個步驟應該具體可執行
3. **標記進度**：使用 [ ] / [x] 追蹤
4. **靈活調整**：根據實際情況修改計畫

## 與其他技能的整合

- **spawn-protocol**：啟動子代理前先規劃
- **kanban-ops**：在看板任務中添加執行計畫
- **task-optimizer**：使用執行計畫驗證複雜度估計

---

**核心價值**：明確的執行計畫可以減少 30-40% 的返工時間，提高 50% 的完成率。
