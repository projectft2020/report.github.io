## 研究摘要

本論文提出了對投影追蹤樹（Projection Pursuit Tree, PPtree）分類器的增強版本，並開發了視覺診斷方法來評估算法改進的效果。原始 PPtree 算法將樹的深度限制為 G-1（G 為類別數），這對複雜分類問題來說過於僵化。研究通過兩種主要擴展改進了算法：（1）通過類別子集化產生更好的決策邊界；（2）允許每個類別進行多次基於熵減少的分割。在 94 個基準數據集的測試中，改進版本在 18 個數據集上優於 CART 和原始 PPtree 算法。研究還開發了互動式 Shiny Web 應用程序和高維可視化工具，用於診斷和解釋算法行為。

## 主要發現

1. **原始 PPtree 的根本限制** — 原始算法的樹結構深度限制為 G-1，每個原始類別必須分配到單個終端節點。這在處理具有不等方差-協方差結構和非線性類別邊界的多類別問題時表現不佳，決策邊界往往過於接近某一個類別，導致不必要的錯誤率。 | 來源: arXiv:2602.21130

2. **算法改進一：類別子集化（Modification 1）** — 該修改不再將所有剩餘類別組合成單個超類別，而是僅使用投影均值差異最小的兩個最近類別來計算下一個投影和定義分割點。這產生更對稱的決策邊界，減少由不等方差-協方差引起的邊界偏移。 | 來源: arXiv:2602.21130

3. **算法改進二：基於熵的多次分割（Modification 2）** — 引入了一種新的方法來選擇分割值，基於結果類別的不純度（負熵）。該方法允許在同一類別內進行多次分割，從而提高了建模非線性類別邊界的靈活性，特別是當類別被分割成多個聚類時。 | 來源: arXiv:2602.21130

4. **互動式視覺診斷工具（Shiny App）** — 開發了一個包含三個模擬標籤頁的 Shiny Web 應用程序（Basic-Sim、Sim-Outlier、MixSim），允許用戶生成不同類型分離的 2D 數據集，並並排比較 rpart、PPtree 和修改後 PPtree 的決策邊界。該工具在算法開發過程中至關重要。 | 來源: arXiv:2602.21130

5. **高維視覺診斷（Tours）** — 使用 tourr 包提供的導向遊覽和切片遊覽來檢查高維數據集的結構和模型擬合。這種方法為性能比較結果提供了解釋，並為為什麼修改後的算法表現更好提供了直覺。 | 來源: arXiv:2602.21130

6. **性能評估結果** — 在 94 個基準數據集上進行測試（每個數據集重複 200 次訓練/測試分割），改進的 PPtree 在 18 個數據集上優於 CART 和原始 PPtree。當組之間的最佳分離位於斜向方向時，優於 CART；當類別方差不同或存在多個類別時，優於原始 PPtree。 | 來源: arXiv:2602.21130

7. **PPtreeExt R 包實現** — 所有改進的算法和互動式視覺診斷工具都在 R 包 PPtreeExt 中實現，CRAN DOI: 10.32614/CRAN.package.PPtreeExt，由 da Silva、Cook 和 Lee 開發（2026）。 | 來源: arXiv:2602.21130

## 詳細分析

### 1. 研究背景與動機

#### 1.1 投影追蹤樹（PPtree）概述

投影追蹤樹算法（Y. D. Lee 等人，2013）是一種創新的分類方法，它結合了樹模型的直觀性和投影追蹤的能力。其核心特徵包括：

- **變量線性組合**：與傳統樹算法使用單個變量的軸對齊分割不同，PPtree 使用變量的線性組合（斜向分割）來進行節點分割
- **投影追蹤指數優化**：優化投影追蹤指數（如 LDA 或 PDA），基於類別信息來識別在每個節點最佳分離組的一維投影
- **緊湊的樹結構**：樹的深度限制為 G-1（G 為類別數），產生緊湊且可解釋的樹
- **無需剪枝**：由於樹結構簡單，不需要進行剪枝操作

#### 1.2 原始 PPtree 的優勢

PPtree 在某些場景下已被證明優於隨機森林（RF），這源於以下優勢：

1. **利用變量相關性**：能夠利用預測變量之間的相關性來改善類別分離
2. **斜向分割能力**：當最佳分離需要變量組合時，能夠更有效地捕捉分離模式
3. **計算效率**：相比於隨機森林中隨機生成的斜向投影，PPtree 的投影追蹤方法更高效

#### 1.3 原始 PPtree 的根本限制

儘管有這些優勢，原始 PPtree 存在嚴重的局限性，這些限制在複雜的實際應用中會顯著影響性能：

**限制 1：僵化的樹深度**
- 深度限制為 G-1，每個原始類別必須分配到單個終端節點
- 當類別具有複雜的內部結構（如被分割成多個聚類）時，這種限制過於僵化
- 無法靈活地處理非線性決策邊界

**限制 2：決策邊界偏移**
- 由於類別分組的方式（將多個類別組合成超類別），預測邊界往往過於接近某一個類別
- 這會導致不必要的錯誤率增加
- 邊界位置的偏移對分類準確性產生顯著影響

**限制 3：方差-協方差假設問題**
- 當類別分組產生不等方差-協方差時，LDA 使用的合併方差-協方差矩陣變得不充分
- 這會影響投影方向的選擇和分割點的確定
- 在異質性數據集中表現不佳

**限制 4：非橢圓形分佈處理**
- 當類別具有非橢圓形方差-協方差時，算法的假設被違反
- 當類別被分割成多個聚類時，G-1 節點結構的剛性不足以靈活處理
- 算法無法適應複雜的數據形狀

### 2. 原始 PPtree 算法詳述

#### 2.1 算法步驟

原始 PPtree 算法遵循一個四步驟流程：

**Step 1: 類別分離**
- 優化投影追蹤指數（如 LDA 或 PDA）
- 找到投影向量 α，最佳地分離當前節點中的所有類別

**Step 2: 二元重新標記**
- 在投影數據 z_i = α^T x_i 上，計算投影類別均值 z̄_1, ..., z̄_G
- 識別具有最大均值差的配對，計算中點 mp
- 根據 z̄_g 是否小於中點 mp，將觀測值重新標記為兩個超類別 g1* 和 g2*

**Step 3: 節點分割**
- 優化新的投影 α*，基於重新標記的類別來分離 g1* 和 g2*
- 使用預定義規則在投影數據上定義分割值 c
- 使用規則 z_i* < c 來分割節點

**Step 4: 遞歸**
- 如果節點包含多個類別，在該節點上重復過程
- 直到所有節點都包含單個類別（深度達到 G-1）

#### 2.2 八種分割規則

原始 PPtree 算法提供了八種不同的規則來計算分割點：

**規則 1: 均值（mean）**
```
c = 1/2 x̄_1 + 1/2 x̄_2
```

**規則 2: 樣本量加權均值（sample size weighted mean）**
```
c = (n_2 / (n_1 + n_2)) x̄_1 + (n_1 / (n_1 + n_2)) x̄_2
```

**規則 3: 標準差加權均值（standard deviation weighted mean）**
```
c = (s_2 / (s_1 + s_2)) x̄_1 + (s_1 / (s_1 + s_2)) x̄_2
```

**規則 4: 標準誤加權均值（standard error weighted mean）**
```
c = (s_2/√n_2 / (s_1/√n_1 + s_2/√n_2)) x̄_1 + (s_1/√n_1 / (s_1/√n_1 + s_2/√n_2)) x̄_2
```

**規則 5: 中位數（median）**
```
c = 1/2 med_1 + 1/2 med_2
```

**規則 6: 樣本量加權中位數（sample size weighted median）**
```
c = (n_2 / (n_1 + n_2)) med_1 + (n_1 / (n_1 + n_2)) med_2
```

**規則 7: IQR 加權中位數（IQR weighted median）**
```
c = (IQR_2 / (IQR_1 + IQR_2)) med_1 + (IQR_1 / (IQR_1 + IQR_2)) med_2
```

**規則 8: 樣本量和 IQR 加權中位數（sample size and IQR weighted median）**
```
c = (IQR_2/√n_2 / (IQR_1/√n_1 + IQR_2/√n_2)) med_1 + (IQR_1/√n_1 / (IQR_1/√n_1 + IQR_2/√n_2)) med_2
```

其中：
- x̄_g：組 g 的均值
- x_g^med：組 g 的中位數
- s_g：組 g 的標準差
- IQR_g：組 g 的四分位距
- n_g：組 g 的樣本量

### 3. 算法改進詳述

#### 3.1 改進一：類別子集化（Subsetting Classes）

##### 核心思想

這個修改針對原始算法的第三步。與其將所有剩餘類別組合成單個超類別，不如僅使用投影均值差異最小的兩個最近類別來計算下一個投影和定義分割點。

##### 算法改進細節

**原始方法**：
- 將 g1* 以外的所有類別組合成單個超類別 g2*
- 使用所有類別的組合信息計算投影和分割點
- 導致分割點向方差較大的組偏移

**改進方法**：
1. 識別投影均值差異最小的兩個類別
2. 從 g1* 中選擇一個類別，從 g2* 中選擇一個類別
3. 僅使用這兩個最近類別的信息來計算新的投影向量 α**
4. 使用預定義規則確定最佳分割點 c

##### 實施步驟

```
3* 找到最優的一維投影 α**，使用子集 {(x_i, y_i*)}_{i=1}^n
   來分離兩類問題 g1* 和 g2*。
   對於這個第二個投影，僅使用來自兩個最接近組的信息，
   一個來自 g1*，另一個來自 g2*。
   g1* 和 g2* 的最佳分離由 α*^T x̄_{g1*} < c 決定；
   如果成立，則將 g1* 分配到左節點，否則將 g2* 分配到右節點，
   其中 x̄_{g1*} 是 g1* 的均值。
```

##### 優勢分析

1. **更對稱的邊界**：決策邊界在群組之間更對稱地放置，避免向某一個類別偏移
2. **減少偏移**：減少由不等方差-協方差引起的邊界位置偏移
3. **更好的多類別處理**：在多類別情況下更好地處理類別間差異
4. **更準確的分割點**：使用最接近的類別對，可以更準確地確定分割位置

#### 3.2 改進二：基於熵的多次分割（Entropy-Based Multiple Splits）

##### 核心思想

這個修改引入了一種新方法來選擇分割值，基於結果類別的不純度。這與 rpart 算法中使用的不純度標準類似，但它操作於投影數據而非單個特徵。

##### 熵作為不純度度量

**負熵公式**：
```
E(s) = -∑_{j=1}^G p_js × log(p_js)
```

其中：
- p_js：子集 s 中來自類別 j 的觀測值比例
- G：類別數
- E(s)：子集 s 的熵

**解釋**：
- 熵值越高表示類別不純度越大
- 當子集中的所有觀測值屬於同一類別時，熵達到最小值 E(s) = 0，表示純節點
- 熵衡量了結果類別的均勻程度

##### 聯合不純度

為評估候選分割的質量，使用左右子集的加權平均熵：

**聯合熵公式**：
```
E(s_L, s_R) = (n_L / (n_L + n_R)) × E(s_L) + (n_R / (n_L + n_R)) × E(s_R)
```

其中：
- n_L, n_R：左右子集中的觀測值數量
- s_L, s_R：左右子集
- E(s_L), E(s_R)：左右子集的熵

##### 算法步驟

```
Step 1: 類別分離
優化投影追蹤指數（如 LDA 或 PDA）找到最佳投影向量 α

Step 2: 投影和候選分割
在投影數據 z_i = α^T x_i 上，定義候選分割為連續 z_i 值之間的中點

Step 3: 不純度評估
對每個候選分割 c：
- 將數據分割為 s_L = {z_i : z_i < c} 和 s_R = {z_i : z_i ≥ c}
- 計算每個子集的熵 E(s_L) 和 E(s_R)
- 計算聯合不純度 E(s_L, s_R)

Step 4: 最佳分割選擇
選擇最小化 E(s_L, s_R) 的分割點 c*
使用規則 z_i < c* 來分割節點
```

##### 特點分析

1. **重大變化**：這代表了對原始 PPtree 結構的重大變化
2. **靈活性增強**：允許在同一類別內進行多次分割，提高建模非線性邊界的靈活性
3. **局部差異關注**：從類別級摘要統計轉移到個體觀測值之間的局部差異
4. **新增停止規則**：為支持這種靈活性，需要新的停止標準
5. **樣本量需求**：與 rpart 類似，但需要更大的每節點樣本量來可靠估計投影係數

### 4. 視覺診斷方法

#### 4.1 互動式 Shiny 應用程序

##### 開發動機

提出算法增強是容易的，理論上修改可能看起來合理。但在實踐中，它們可能無法按預期工作。因此，開發算法修改的一個重要部分是提供驗證，證明它們按預期執行。這意味著超越預測準確性，理解結果模型。

##### 應用程序特點

**三個模擬標籤頁**：

1. **Basic-Sim**：
   - 模擬良好分離的類別
   - 提供基準比較場景

2. **Sim-Outlier**：
   - 包含離群值聚類
   - 測試算法對異常值的魯棒性

3. **MixSim**：
   - 使用 MixSim 包生成重疊群組
   - 測試算法在複雜邊界情況下的表現

**參數控制**：
- 允許用戶調整數據生成參數
- 控制算法參數（如分割規則選擇）
- 實時更新決策邊界可視化

**並排比較**：
- 同時顯示三種算法的決策邊界：
  - rpart（CART 實現）
  - 原始 PPtree
  - 修改後的 PPtree
- 使用不同顏色區分類別
- 清晰顯示分割線和邊界

##### 應用價值

1. **算法調試**：
   - 識別數據中不同方法表現更好的模式
   - 快速驗證算法改進的有效性
   - 發現潛在的實施錯誤

2. **邊界視覺化**：
   - 直接比較不同算法產生的決策邊界
   - 直觀理解算法行為差異
   - 評估邊界的對稱性和合理性

3. **敏感性分析**：
   - 了解數據中的小變化如何影響擬合模型
   - 特別是在涉及投影時的敏感性
   - 識別算法的魯棒性

4. **教育用途**：
   - 幫助用戶理解不同分類算法的差異
   - 直觀展示斜向分割的優勢
   - 解釋投影追蹤的概念

#### 4.2 高維可視化（Tours）

##### 技術基礎

使用 tourr 包（Wickham 等人，2011）提供的投影遊覽技術：

**導向遊覽（Guided Tour）**：
- 根據優化目標（如 LDA 指數）選擇投影基底
- 尋找最佳展示類別分離的視角
- 系統地探索高維空間

**切片遊覽（Slice Tour）**：
- 專注於數據域中心的邊界
- 通過切片展示邊界結構
- 減少視覺混亂

##### 診斷策略

**策略 1：在數據上工作**

a. **兩兩處理**：
   - 如果有超過 2 個組，一次處理兩個組
   - 簡化可視化複雜度

b. **導向遊覽**：
   - 在數據上使用導向遊覽
   - 找到變量組合中群組分離的地方
   - 識別最佳分離方向

**策略 2：預測邊界**

a. **生成邊界點**：
   - 預測覆蓋數據域的網格值
   - 對感興趣的方法（PPtreeExt、rpart、PPtree）
   - 提取靠近分類器生成邊界的點

b. **記錄投影序列**：
   - 記錄顯示 PPtree 創建邊界的投影基底序列
   - 保存投影變化的軌跡

c. **比較邊界**：
   - 使用相同的投影序列查看三種不同方法的邊界
   - 使用切片遊覽專注於數據域中心的邊界
   - 對比不同算法的邊界行為

#### 5.1 基準數據集研究設計

##### 數據集規模與來源

**數據集規模**：

##### 典型案例深入分析

**案例 1：data20-3**
- **特徵**：8143 觀測值，5 個預測變量，2 個類別
- **優勢原因**：
  - 組 0 被分割在組 1 的兩側
  - 原始 PPtree 無法捕獲這種結構（每個原始類別必須分配到單個終端節點）
  - 分離主要在 V3 上，但其他變量的少量幫助改進了分離
- **改進效果**：
  - 改進 2 允許多次分割，能夠捕獲組 0 的兩個聚類
  - 修改後的算法能夠更好地處理這種結構

**案例 2：data49-2**
- **特徵**：606 觀測值，100 個預測變量，2 個類別
- **優勢原因**：
  - 每個類別顯示三個指向不同方向的線性片段
  - 高維度使得可視化具有挑戰性
  - 使用 PCA 降維後，PC2 和 PC3 顯示有趣的類別差異
- **改進效果**：
  - 原始 PPtree 由於單次分割限制無法捕獲複雜結構
  - 改進 2 允許多次分割，能夠處理多個線性片段
  - 在變量組合中，類別幾乎可以通過多個線性切割分離

**案例 3：data69-1**
- **特徵**：5000 觀測值，21 個預測變量，3 個類別
- **優勢原因**：
  - 三個類別大小相似，但在 2D 投影中形成三角形
  - 需要斜向切割才能有效分離
- **改進效果**：
  - 改進 1 產生更對稱的決策邊界
  - 改進 2 允許非線性邊界的靈活建模
  - 修改後的算法處理方差差異比原始算法更好

#### 6.3 應用場景

**適合使用改進 PPtree 的場景**：

1. **高維數據分類問題**：
   - 變量數目大於觀測值數目（p > n）
   - 需要降維和特徵選擇

2. **變量相關性強**：
   - 變量之間存在強相關性
   - 最佳分離需要變量組合

3. **斜向分離優勢**：
   - 類別之間的最佳分離位於斜向方向
   - 軸對齊分割效率低下

4. **不等方差-協方差**：
   - 不同類別具有不同的方差-協方差結構
   - 原始 PPtree 的邊界偏移問題

5. **多類別問題**：
   - 類別數較多（> 2）
   - 需要靈活的類別分組策略

6. **非線性決策邊界**：
   - 類別被分割成多個聚類
   - 需要多次分割來捕獲複雜結構

7. **需要可解釋性**：
   - 模型決策需要解釋和驗證
   - 醫學診斷、金融風險評估等領域

### 7. 應用價值評估

#### 7.1 優勢

1. **改進的分類性能**：
   - 在 94 個基準數據集中的 18 個上優於 CART 和原始 PPtree
   - 特別是在處理斜向分離和多類別問題時表現突出
   - 在某些數據集上甚至優於集成方法

2. **靈活性顯著增強**：
   - 通過允許多次分割，打破了 G-1 的深度限制
   - 通過類別子集化，實現了更靈活的類別分組
   - 能夠處理更複雜的非線性決策邊界

3. **保持可解釋性**：
   - 保留了樹模型的直觀性和可解釋性
   - 決策規則清晰易於理解
   - 適合需要模型解釋的應用場景

4. **完整的視覺診斷工具**：
   - 提供互動式 Shiny 應用程序
   - 支持高維可視化（tours）
   - 使得算法行為的調試和評估變得直觀高效

5. **可擴展性強**：
   - 修改後的樹算法可以整合到投影追蹤森林分類器中
   - 利用 bagging 可以進一步改進非線性決策邊界問題中的性能和穩健性
   - 有內建的變量選擇機制

6. **開源且易於使用**：
   - 作為 R 包提供，便於研究人員和實踐者使用
   - 文檔完整，包含示例和教程
   - 社區支持和持續改進

7. **理論基礎紮實**：
   - 基於投影追蹤理論
   - 結合了統計學和機器學習的優勢
   - 有完整的算法描述和證明

#### 7.2 局限性

1. **性能提升有限**：
   - 在 94 個數據集中僅在 18 個上表現優於 CART 和原始 PPtree
   - 約占 19%，性能提升的普遍性有待驗證
   - 在多數數據集上，改進版本的優勢不明顯

2. **計算複雜度較高**：
   - 投影追蹤需要優化投影指數，計算成本高
   - 與簡單的軸對齊分割相比，計算時間顯著增加
   - 基於熵的多次分割方法需要評估所有候選分割點，進一步增加計算負擔

3. **樣本量需求較大**：
   - 投影係數的可靠估計需要較大的每節點樣本量
   - 可能限制在小數據集上的應用
   - 對於 p >> n 的情況（高維小樣本），性能可能受限

4. **高維可視化挑戰**：
   - 雖然提供了視覺診斷工具，但在非常高維的數據集上（如 617 個變量的 ISOlet 數據集），可視化仍然具有挑戰性
   - 即使使用 PCA 降維，也可能遺失重要信息
   - 用戶需要一定的統計學知識來解釋高維可視化

5. **超參數敏感性**：
   - 算法的性能對選擇的投影指數（LDA、PDA 等）敏感
   - 對分割規則（8 種不同規則 + 熵基規則）的選擇敏感
   - 缺乏自動化的超參數選擇方法

6. **集成方法的競爭**：
   - 在 3 個案例中，集成分類器（如隨機森林、PPforest）仍然表現最好
   - 這表明在某些情況下，單樹方法可能仍然受限
   - 改進的樹作為基學習器的價值需要在集成方法中進一步驗證

7. **非橢圓分佈處理能力有限**：
   - 雖然改進了靈活性，但對於極度非橢圓形或高度異質的數據分佈，算法可能仍不足夠
   - 算法仍然基於投影追蹤的假設（尋找最佳線性投影）
   - 對於高度非線性或複雜流形數據，可能需要其他方法

8. **缺乏大規模數據擴展**：
   - 當前實現可能不適合處理超大型數據集（數百萬觀測值）
   - 缺乏增量學習或分布式計算支持
   - 對於實時應用，計算時間可能過長
