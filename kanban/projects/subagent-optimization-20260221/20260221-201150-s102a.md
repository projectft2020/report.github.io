## Analysis

### Architecture & Design

**Core Identity:**
- **Specialty:** Web research, fact-finding, information synthesis, cross-validation
- **Personality:** Precise, objective, thorough — always cites sources
- **Role:** Sub-agent spawned by Main Orchestrator (Charlie)
- **Independence:** Works independently, returns results via announce, no direct user interaction

**Position in Multi-Agent Ecosystem:**
- **Entry point** in research workflows (Research → Analyst → Creative → Automation)
- **Upstream provider** of raw research data to Analyst Agent
- **Complements** Scout Agent (Scout discovers topics → Research investigates)
- **Separation of concerns:** Research collects info, Analyst analyzes, Creative generates ideas

### Workflow & Process

**7-Step Research Workflow:**
1. **Parse task** — Identify what's being researched
2. **Memory first** — `memory_search` before web search (cost optimization, duplicate prevention)
3. **Research** — `web_search` for discovery, `web_fetch` for depth
4. **Cross-validate** — Confirm important claims with 2+ sources (quality gate)
5. **Synthesize** — Organize findings into clear, structured content
6. **Write output** — Complete result to OUTPUT PATH
7. **Announce** — Brief summary (2-3 sentences)

**Key Design Strengths:**

| Strength | Description | Impact |
|----------|-------------|--------|
| Memory-first | Checks shared knowledge before web searches | Reduces duplicate research, saves API costs |
| Cross-validation | Requires 2+ sources for important claims | Improves reliability, reduces misinformation |
| Structured output | Consistent markdown format with metadata | Enables downstream agents to parse reliably |
| Confidence scoring | Explicit confidence levels (high/medium/low) | Enables downstream decision-making |
| Source attribution | Every finding cites source URL or "memory" | Traceability, reproducibility |

### Tool Permissions & Constraints

**Available Tools:**
- ✅ `web_search` — Search the internet
- ✅ `web_fetch` — Fetch specific pages
- ✅ `read` — Read local files
- ✅ `memory_search` — Search shared knowledge base
- ✅ `write` — Write output files

**Denied Tools:**
- ❌ `exec` — Cannot execute system commands
- ❌ `edit` — Cannot modify files (read-only)
- ❌ `cron` — Cannot schedule recurring tasks

**Design Rationale:**
- Read-only constraint ensures Research Agent focuses on information collection, not modification
- No system command access prevents accidental damage
- Allows writing output files to deliver research results
