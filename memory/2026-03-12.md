### Scout 掃描結果

- ❌ Reddit：401 錯誤（認證問題）
- ❌ Browser Crawler：0 主題（所有任務都已存在）

### Reddit API 認證問題

**問題發現：**
- Reddit Adapter 返回 401 錯誤
- PREFERENCES_v2.json 中缺少 `client_id` 和 `client_secret` 配置
- 從 v2.0 部署開始（2026-03-04）就沒有配置過

**解決方案：**
需要用戶在 Reddit 創建應用獲取憑證：
1. 訪問 https://www.reddit.com/prefs/apps
2. 創建應用（"script" 類型）
3. 獲取 `client_id` 和 `client_secret`
4. 更新 PREFERENCES_v2.json

**配置格式：**
```json
{
  "sources": {
    "reddit_r_quant": {
      "config": {
        "subreddit": "quant",
        "client_id": "YOUR_CLIENT_ID",
        "client_secret": "YOUR_CLIENT_SECRET",
        "user_agent": "Scout v2.0"
      }
    },
    "reddit_r_algotrading": {
      "config": {
        "subreddit": "algotrading",
        "client_id": "YOUR_CLIENT_ID",
        "client_secret": "YOUR_CLIENT_SECRET",
        "user_agent": "Scout v2.0"
      }
    }
  }
}
```

**影響：**
- Reddit 數據源暫時無法使用
- r/quant 和 r/algotrading 社群內容無法掃描
- 影響研究主題發現（約 20-30% 的潛在來源）

**優先級：** P2（中等，不影響核心功能）

## 14:17 - Telegram 私人對話設置

### 推薦配置
```json
{
  "allowBots": true,
  "requireMention": false
}
```

### 關鍵點
- `requireMention: false` 是必須的！
  - Telegram 私人對話不支持 @ 功能
  - 如果設為 `true`，機器人永遠不會回應
  - 私人對話中機器人應該自動回應所有訊息

- `allowBots: true` 是安全的
  - 私人對話中很少有其他機器人
  - 即使有，看到也沒問題

## 15:38 - 自動改進 Daemon 實現

### 決策
用戶選擇優先級：「1：自動改進 daemon」

這是正確的優先級 - 技術基礎先打好，之後的商業化和平台化才能順利進行。

### 實現內容

#### 1. 核心腳本：`kanban-ops/auto_improve_daemon.py`
- **功能：** 類似 Felix 的 cron job 系統
- **機制：** 每天（或間隔 24 小時）自動執行改進循環
- **大小：** 14,409 bytes（完整實現）

**核心流程：**
```
收集最近 24 小時的 session files
    ↓
分析錯誤模式、重複任務、新知識點
    ↓
找出 1 個改進點（按優先級）
    ↓
應用改進（修改 .md 文件）
    ↓
記錄改進日誌
    ↓
更新下次改進時間
```

**分析重點：**
1. **錯誤模式：** timeout, failed, error, exception, 卡住, stuck, terminated
2. **重複任務：** 研究、分析、回測、測試、檢查
3. **新知識點：** 學到、發現、洞察、insight、關鍵、重要、核心

**改進類型：**
| 類型 | 優先級 | 說明 |
|------|--------|------|
| `fix_error` | High | 修復錯誤模式 |
| `record_insight` | Medium | 記錄新知識點 |
| `optimize_workflow` | Medium | 優化工作流 |
| `update_documentation` | Low | 更新文檔 |

#### 2. 配置文件：`kanban-ops/AUTO_IMPROVE.json`
```json
{
  "daemon": {
    "name": "auto_improve",
    "version": "1.0.0",
    "enabled": true,
    "schedule": {
      "cron": "0 2 * * *",
      "timezone": "Asia/Taipei",
      "description": "每天凌晨 2 點執行"
    }
  },
  "improvement": {
    "max_per_day": 1,
    "min_hours_between": 24,
    "analysis_window_hours": 24
  }
}
```

#### 3. 文檔：`kanban-ops/AUTO_IMPROVE_README.md`
- 完整的使用說明
- 配置指南
- 複利效果計算
- 未來改進路線圖
- 最佳實踐

```python
def backup_target_file(self, target_file: str) -> bool:
    """
    備份目標文件

    Args:
        target_file: 目標文件路徑

    Returns:
        是否成功
    """
    try:
        # 解析相對路徑
        if not target_file.startswith("/"):
            target_path = self.workspace / target_file
        else:
            target_path = Path(target_file)

        if not target_path.exists():
            print(f"   ℹ️  目標文件不存在，跳過備份: {target_file}")
            return True

        # 創建備份目錄
        backup_dir = self.workspace / ".backups" / "improvements"
        backup_dir.mkdir(parents=True, exist_ok=True)

        # 生成備份文件名
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        backup_name = f"{target_path.name}.{timestamp}.backup"
        backup_path = backup_dir / backup_name

        # 複製文件
        import shutil
        shutil.copy2(target_path, backup_path)
        print(f"   ✅ 備份成功: {backup_path}")

        return True
    except Exception as e:
        print(f"   ❌ 備份失敗: {e}")
        return False
```

##### 2. 回滾機制
```python
def rollback_improvement(self, improvement: Dict) -> bool:
    """
    回滾改進

    Args:
        improvement: 改進建議

    Returns:
        是否成功
    """
    try:
        # 解析相對路徑
        target_file = improvement["target_file"]
        if not target_file.startswith("/"):
            target_path = self.workspace / target_file
        else:
            target_path = Path(target_file)

        # 找到最新的備份
        backup_dir = self.workspace / ".backups" / "improvements"
        backups = sorted(backup_dir.glob(f"{target_path.name}.*.backup"), reverse=True)

        if not backups:
            print(f"   ⚠️  沒有找到備份文件")
            return False

        # 恢復最新的備份
        latest_backup = backups[0]
        import shutil
        shutil.copy2(latest_backup, target_path)
        print(f"   ✅ 回滾成功: {latest_backup}")

        return True
    except Exception as e:
        print(f"   ❌ 回滾失敗: {e}")
        return False
```

##### 3. 驗證機制
```python
def validate_improvement(self, improvement: Dict) -> bool:
    """
    驗證改進

    檢查改進後的文件是否有效

    Args:
        improvement: 改進建議

    Returns:
        是否驗證通過
    """
    try:
        # 解析相對路徑
        target_file = improvement["target_file"]
        if not target_file.startswith("/"):
            target_path = self.workspace / target_file
        else:
            target_path = Path(target_file)

        # 檢查文件是否存在
        if not target_path.exists():
            print(f"   ❌ 驗證失敗: 文件不存在")
            return False

        # 檢查文件大小
        file_size = target_path.stat().st_size
        if file_size == 0:
            print(f"   ❌ 驗證失敗: 文件為空")
            return False

        # 檢查文件是否為有效的 Markdown（如果是 .md 文件）
        if target_path.suffix == ".md":
            content = target_path.read_text(encoding="utf-8")
            if len(content) < 10:
                print(f"   ❌ 驗證失敗: 文件內容太少")
                return False

        print(f"   ✅ 驗證通過: 文件大小 {file_size} bytes")
        return True
    except Exception as e:
        print(f"   ❌ 驗證失敗: {e}")
        return False
```

##### 4. 應用改進（更新）
```python
def apply_improvement(self, improvement: Dict) -> bool:
    """
    應用改進

    修改系統文件（.md, .py）或啟動 Agent 處理

    Args:
        improvement: 改進建議

    Returns:
        是否成功
    """
    print(f"\n🔧 應用改進...")
    print(f"   類型: {improvement['type']}")
    print(f"   目標: {improvement['target_file']}")
    print(f"   變更: {improvement['change']}")
    print(f"   原因: {improvement['reason']}")
    print(f"   優先級: {improvement['priority']}")

    # 備份目標文件 ✨ 新增
    backup_success = self.backup_target_file(improvement["target_file"])
    if not backup_success:
        print(f"⚠️  備份失敗，跳過改進")
        self.log_improvement(improvement, status="backup_failed")
        return False

    # 根據改進類型執行不同的操作
    success = False
    if improvement["type"] == "record_insight":
        success = self.apply_insight_improvement(improvement)
    elif improvement["type"] == "fix_error":
        success = self.apply_error_improvement(improvement)
    else:
        print(f"⚠️  未知的改進類型: {improvement['type']}")
        self.log_improvement(improvement, status="unknown_type")
        return False

    # 驗證改進 ✨ 新增
    if success and improvement["type"] == "fix_error":
        validation_success = self.validate_improvement(improvement)
        if not validation_success:
            print(f"⚠️  驗證失敗，回滾改進")
            self.rollback_improvement(improvement)  # ✨ 新增
            self.log_improvement(improvement, status="validation_failed")
            return False

    return success
```
