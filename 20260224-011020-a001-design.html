1. 系統概述¶

1.1 項目目標¶

設計並實現一個獨立的監控服務，用於監控和管理 OpenClaw 工作區的任務、子代理和系統指標。該 Dashboard 將提供：

  • 實時監控：任務狀態、代理活動、系統健康
  • 操作界面：任務管理、代理控制、系統配置
  • 數據可視化：統計圖表、趨勢分析、性能指標
  • RESTful API：完整的 CRUD 接口，支持外部集成

1.2 用戶場景¶

主要用戶： - 個人開發者（Charlie） - 需要監控 OpenClaw 工作區的所有活動

核心場景：

  1. 日常監控
  2. 查看任務統計（總數、待辦、進行中、已完成）
  3. 查看代理狀態（活躍、空閒、錯誤）

  4. 檢查系統健康（Scout 統計、心跳狀態）

  5. 任務管理

  6. 創建新任務
  7. 查看任務詳情和日誌
  8. 批量操作（重新啟動、取消、刪除）

  9. 任務依賴關係可視化

  10. 子代理監控

  11. 查看所有子代理的實時狀態
  12. 查看代理日誌和輸出
  13. 健康檢查和診斷

  14. 代理資源使用情況

  15. 系統設置

  16. Scout 掃描配置
  17. 心跳監控設置
  18. 系統通知配置
  19. 日誌級別管理

4.4.1 Phase 1：JSON 文件存儲¶

import json
from pathlib import Path
from typing import List, Optional
from datetime import datetime

class TaskRepository:
    """任務數據存儲庫（基於 JSON）"""

    def __init__(self, file_path: Path):
        self.file_path = file_path

    def load_all(self) -> List[dict]:
        """加載所有任務"""
        if not self.file_path.exists():
            return []

        with open(self.file_path, 'r', encoding='utf-8') as f:
            return json.load(f)

    def save_all(self, tasks: List[dict]):
        """保存所有任務"""
        with open(self.file_path, 'w', encoding='utf-8') as f:
            json.dump(tasks, f, indent=2, ensure_ascii=False)

    def get_by_id(self, task_id: str) -> Optional[dict]:
        """根據 ID 獲取任務"""
        tasks = self.load_all()
        for task in tasks:
            if task.get('id') == task_id:
                return task
        return None

    def create(self, task: dict) -> dict:
        """創建任務"""
        tasks = self.load_all()
        tasks.append(task)
        self.save_all(tasks)
        return task

    def update(self, task_id: str, updates: dict) -> Optional[dict]:
        """更新任務"""
        tasks = self.load_all()
        for i, task in enumerate(tasks):
            if task.get('id') == task_id:
                tasks[i].update(updates)
                tasks[i]['updated_at'] = datetime.now().isoformat()
                self.save_all(tasks)
                return tasks[i]
        return None

    def delete(self, task_id: str) -> bool:
        """刪除任務"""
        tasks = self.load_all()
        new_tasks = [t for t in tasks if t.get('id') != task_id]
        if len(new_tasks) < len(tasks):
            self.save_all(new_tasks)
            return True
        return False

    def filter(self, filters: dict) -> List[dict]:
        """過濾任務"""
        tasks = self.load_all()
        result = tasks

        if 'status' in filters:
            result = [t for t in result if t.get('status') == filters['status']]
        if 'agent' in filters:
            result = [t for t in result if t.get('agent') == filters['agent']]
        if 'priority' in filters:
            result = [t for t in result if t.get('priority') == filters['priority']]

        return result

interface TaskContextType {
  tasks: Task[];
  loading: boolean;
  error: string | null;
  fetchTasks: () => Promise<void>;
  createTask: (task: TaskCreate) => Promise<Task>;
  updateTask: (id: string, updates: TaskUpdate) => Promise<void>;
  deleteTask: (id: string) => Promise<void>;
}

const TaskContext = createContext<TaskContextType | undefined>(undefined);

export const TaskProvider: React.FC<{ children: ReactNode }> = ({ children }) => {
  const [tasks, setTasks] = useState<Task[]>([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const fetchTasks = async () => {
    setLoading(true);
    setError(null);
    try {
      const response = await fetch('/api/v1/tasks');
      const data = await response.json();
      setTasks(data.tasks);
    } catch (err) {
      setError('Failed to fetch tasks');
    } finally {
      setLoading(false);
    }
  };

  const createTask = async (task: TaskCreate) => {
    const response = await fetch('/api/v1/tasks', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(task),
    });
    const newTask = await response.json();
    setTasks(prev => [...prev, newTask]);
    return newTask;
  };

  const updateTask = async (id: string, updates: TaskUpdate) => {
    const response = await fetch(`/api/v1/tasks/${id}`, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(updates),
    });
    const updatedTask = await response.json();
    setTasks(prev => prev.map(t => t.id === id ? updatedTask : t));
  };

  const deleteTask = async (id: string) => {
    await fetch(`/api/v1/tasks/${id}`, { method: 'DELETE' });
    setTasks(prev => prev.filter(t => t.id !== id));
  };

  return (
    <TaskContext.Provider
      value={{
        tasks,
        loading,
        error,
        fetchTasks,
        createTask,
        updateTask,
        deleteTask,
      }}
    >
      {children}
    </TaskContext.Provider>
  );
};

8.6 故障排除¶

8.6.1 常見問題¶

問題：API 返回 404 - 檢查 URL 路徑是否正確 - 檢查 FastAPI 服務器是否運行 - 檢查端口是否正確（默認 8001）

問題：前端無法訪問後端 - 檢查 VITE_API_URL 環境變量 - 檢查後端服務器是否運行 - 檢查 CORS 配置

問題：文件監控不工作 - 檢查文件路徑是否正確掛載 - 檢查 watchdog 庫是否正確安裝 - 檢查文件權限

問題：Docker 容器無法啟動 - 檢查 Dockerfile 語法 - 檢查端口是否被占用 - 檢查環境變量是否設置
