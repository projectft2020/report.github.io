## Detailed Analysis

### Package Overview

genriesz is a specialized Python package designed for automatic debiased machine learning with Generalized Riesz Regression (GRR) under Bregman divergences. The package was developed by Masahiro Kato and is available under the GNU General Public License v3.0. It addresses the challenge of estimating causal and structural parameters by unifying various debiasing methods into a single cohesive framework.

The package is available through PyPI (`pip install genriesz`) and includes optional integrations with scikit-learn and PyTorch for extended functionality. It requires Python 3.10 or higher and has dependencies on NumPy and SciPy for core functionality.

### Debiasing Methods

genriesz implements several debiasing methods unified under the framework of Generalized Riesz Regression:

**1. Generalized Riesz Regression (GRR):**
- Estimates Riesz representers by minimizing empirical Bregman divergences
- Unifies various methods including covariate balancing, nearest-neighbor matching, calibrated estimation, and density ratio estimation
- Provides a systematic approach to debiased estimation with theoretical guarantees

**2. Automatic Regressor Balancing (ARB):**
- Key innovation that automatically constructs compatible link functions
- Given a Bregman generator and representer model class, ARB ensures balancing optimality conditions
- Eliminates the need for manual specification of Riesz representer analytic forms

**3. Multiple Estimation Approaches:**
- **Regression Adjustment (RA)**: $\hat{\theta}^{RA} = \frac{1}{n}\sum_{i=1}^{n} m(W_i, \hat{\gamma})$
- **Riesz Weighting (RW)**: $\hat{\theta}^{RW} = \frac{1}{n}\sum_{i=1}^{n} \hat{\alpha}(X_i) Y_i$
- **Augmented Riesz Weighting (ARW)**: $\hat{\theta}^{ARW} = \frac{1}{n}\sum_{i=1}^{n} [\hat{\alpha}(X_i)(Y_i - \hat{\gamma}(X_i)) + m(W_i, \hat{\gamma})]$
- **TMLE-style**: Uses targeted maximum likelihood estimation with fluctuation updates

**4. Bregman Divergence Framework:**
The package supports various Bregman generators, each corresponding to different debiasing methods:

| Bregman Generator | Density Ratio Estimation | Riesz Representer Estimation |
|-------------------|-------------------------|------------------------------|
| $(\alpha - C)^2$ | LSIF/KuLSIF | SQ-Riesz regression |
| $(|\alpha| - C)\log(|\alpha| - C) - |\alpha|$ | UKL divergence | UKL-Riesz regression |
| $(|\alpha| - C)\log(|\alpha| - C) - (|\alpha| + C)\log(|\alpha| + C)$ | BKL divergence | BKL-Riesz regression |
| Basu's Power divergence | BP divergence | BP-Riesz regression |
| PU learning loss | PU learning | PU-Riesz regression |

### Implementation Details

**Core Architecture:**
- The main entry point is `grr_functional()` for generic linear functionals
- Convenience wrappers include `grr_ate()`, `grr_att()`, `grr_did()`, and `grr_ame()`
- Results objects store point estimates, standard errors, confidence intervals, and p-values
- Supports cross-fitting with user-specified number of folds

**Basis Function System:**
- **PolynomialBasis**: Polynomial features with configurable degree
- **RBFRandomFourierBasis**: Random Fourier features for RKHS approximation
- **RBFNystromBasis**: Nyström approximation for kernel methods
- **KNNCatchmentBasis**: Nearest-neighbor matching expressed as catchment areas
- **RandomForestLeafBasis**: Scikit-learn random forest leaf encodings (optional)
- **Neural network features**: PyTorch-based neural embeddings (optional)

**Optimization and Regularization:**
- Supports $\ell_p$ penalties for any $p \geq 1$
- Uses L-BFGS-B via SciPy with smooth approximation of $\ell_1$ subgradient
- Implements cross-validation for regularization parameter selection
- Provides both primal and dual optimization perspectives

**Handling Domain Constraints:**
- Some generators require constraints (e.g., $|\alpha| > C$ or $\alpha \in (0,1)$)
- Exposes `branch_fn` interface for selecting valid branches (e.g., treated vs. control)
- Ensures fitted representers respect generator domains by construction

### Usage Examples

**1. Basic ATE Estimation:**
```python
import numpy as np
from genriesz import (
    grr_ate, UKLGenerator, 
    PolynomialBasis, TreatmentInteractionBasis
)

# Prepare data: X = [D, Z] where D is treatment, Z are covariates
n, d_z = 1000, 5
rng = np.random.default_rng(0)
Z = rng.normal(size=(n, d_z))
D = (rng.normal(size=n) > 0).astype(float)
Y = 2.0 * D + Z[:, 0] + rng.normal(size=n)
X = np.column_stack([D, Z])

# Define basis and generator
psi = PolynomialBasis(degree=2)
phi = TreatmentInteractionBasis(base_basis=psi)
gen = UKLGenerator(C=1.0, branch_fn=lambda x: int(x[0] == 1.0)).as_generator()

# Estimate ATE
res = grr_ate(
    X=X, Y=Y,
    basis=phi,
    generator=gen,
    cross_fit=True, folds=5,
    riesz_penalty="l2", riesz_lam=1e-3,
    estimators=("ra", "rw", "arw", "tmle"),
)
print(res.summary_text())
```

**2. General Functional Estimation:**
```python
from genriesz import grr_functional, BregmanGenerator

def m(x, gamma):
    # Custom linear functional
    return gamma(x)

def g(x, alpha):
    # Custom Bregman generator
    return 0.5 * alpha**2

def basis(X):
    # Custom basis function
    return np.c_[np.ones(len(X)), X]

# Estimate using general functional
generator = BregmanGenerator(g=g)
res = grr_functional(
    X=X, Y=Y,
    m=m,
    basis=basis,
    generator=generator,
    estimators=("rw",),
)
```
