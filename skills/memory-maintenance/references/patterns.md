# 記憶維護模式

> **參考文件：** 記憶維護中常用的模式和框架
> **用途：** 指導知識提取、分類、整理

---

## 知識提取模式

### 模式 1: 學習點識別

**標記格式：**
- `### 我學到` - 今天學到的新知識
- `### What I've Learned` - 學習總結
- `### 學習總結` - 系統性學習記錄

**提取規則：**
- 1. 掃描 daily log 中的標記
- 2. 提取標記後的內容
- 3. 按日期分組
- 4. 過濾重複內容

**範例：**
```markdown
### 我學到

**背壓機制（Backpressure）：**
- 可複用模式：監測 → 計算健康度 → 動態調整
- 健康度公式：1 - (卡住任務數 / 並發上限)
```

---

### 模式 2: 模式識別

**標記格式：**
- `### 核心模式` - 可重用的框架
- `### 關鍵洞察` - 深度理解
- `### 可複用` - 可直接應用的模式

**提取規則：**
- 1. 識別模式標記
- 2. 提取模式描述
- 3. 按類型分類（系統、研究、工具）
- 4. 記錄應用場景

**範例：**
```markdown
### 核心模式

**背壓機制 = 監測 + 健康度 + 動態調整**
- 監測：檢查卡住任務數、健康度
- 健康度：1 - (卡住任務數 / 並發上限)
- 動態調整：根據健康度調整啟動頻率和並發上限
- 應用場景：任何需要動態調整的系統
```

---

### 模式 3: 決策記錄

**標記格式：**
- `### 關鍵決策` - 重要決策和理由
- `### 重要決策` - 重大方向調整
- `### Key Decisions` - 決策記錄

**提取規則：**
- 1. 識別決策標記
- 2. 提取決策背景、選項、理由、結果
- 3. 按時間排序
- 4. 記錄影響範圍

**範例：**
```markdown
### 關鍵決策

**優化方案選擇：修補 vs 重構**

**背景：**
- 現有系統存在多個問題
- 需要決定是快速修補還是重構

**選項：**
- A. 修補：快速但問題會重複
- B. 重構：慢但一勞永逸

**決策：** 重構（方案 B）

**理由：**
- 長期來看更節省時間
- 技術債累積會更嚴重
- 需要建立 SOP 模式

**結果：**
- 問題一次性解決
- 建立可複用的設計模式
```

---

### 模式 4: 成就記錄

**標記格式：**
- `### 完成項目` - 項目完成
- `### 成就` - 重要成就
- `### 實證成果` - 可量化的成果

**提取規則：**
- 1. 識別成就標記
- 2. 提取成果數據和指標
- 3. 對比修改前後
- 4. 記錄關鍵數字

**範例：**
```markdown
### 實證成果

**系統優化 Phase 1 完成成果：**

| 指標 | 修改前 | 修改後 | 改善 |
|------|--------|--------|------|
| 卡住任務數 | 6 個 | 0 個 | ⬇️ 100% |
| 回滾時間 | 120 分鐘 | 45 分鐘 | ⬇️ 62.5% |
| 健康度監控 | ❌ | ✅ | ✅ 新增 |

**關鍵數字：**
- 完成優化：P0 + P1 + P2（全部完成）
- 研究完成：19 篇
- 系統健康度：1.00（完全健康）
```

## 記憶分類模式

### 分類層次

```
MEMORY.md (主索引，快速導航)
├── Recent Achievements (最近成就)
├── System Status (系統狀態)
├── Key Decisions (關鍵決策)
└── Quick Commands (快捷指令)

memory/topics/ (按主題分類的深度知識)
├── system-architecture.md (系統架構)
├── quantitative-research.md (量化研究)
├── risk-management.md (風險管理)
├── scout-system.md (Scout 系統)
├── monitoring.md (監控系統)
└── user-preferences.md (用戶偏好)

memory/YYYY-MM-DD.md (完整的每日記錄)
├── 原始記錄（完整保留）
├── 學習點（提取後整理）
├── 模式和洞察（識別後歸類）
└── 決策和行動（記錄關鍵決策）
```

## 記憶整理模式

### 規則 1: 去重

**原則：** 同一知識只記錄在最適合的地方

**執行：**
- 檢查新知識是否已存在
- 如果存在，更新舊記錄而不是新建
- 使用相同日期標記更新

**範例：**
```
知識："背壓機制"

✅ 記錄在: memory/topics/system-architecture.md（技術細節）
❌ 不重複記錄在: MEMORY.md（主索引只包含摘要）
✅ 更新: MEMORY.md 添加簡短引用
```

---

### 規則 2: 分層

**原則：** 摘要在上層，細節在下層

**執行：**
- MEMORY.md：只保留摘要和引用
- topics/：保留詳細內容
- daily logs：保留完整原始記錄

**範例：**
```
MEMORY.md:
  ✅ "背壓機制已優化，參考 memory/topics/system-architecture.md"

memory/topics/system-architecture.md:
  ✅ 完整的背壓機制實現細節
  ✅ 健康度計算公式
  ✅ 動態調整規則

memory/2026-03-05.md:
  ✅ 完整的優化過程記錄
  ✅ 原始數據和日誌
```

---

### 規則 3: 時效性

**原則：** 舊記憶定期清理，保持精簡

**執行：**
- 保留最近 30 天的 daily logs
- 超過 30 天的詳細記錄歸檔
- topics/ 中的核心知識永久保留

**範例：**
```
保留:
  ✅ 最近 30 天的 daily logs
  ✅ 所有 topics/ 文件
  ✅ MEMORY.md 和 SOUL.md

清理:
  ❌ 30 天前的詳細日誌
  ❌ 過時的技術細節
  ❌ 已解決的問題記錄
```

## 常見問題

### Q1: 如何處理重複知識？

**A:** 檢查新知識是否已存在：
- 如果存在 → 更新舊記錄
- 如果不存在 → 新建記錄
- 保留最新的時間戳

---

### Q2: 如何決定知識的層次？

**A:** 使用「重要性和細節」原則：
- **重要 + 詳細** → topics/
- **重要 + 摘要** → MEMORY.md
- **個人成長** → SOUL.md
- **完整記錄** → daily logs

---

### Q3: 何時清理舊記憶？

**A:** 定期清理規則：
- **Daily logs:** 保留 30 天
- **Topics:** 永久保留（除非過時）
- **MEMORY.md:** 持續更新，只保留最近成就
- **SOUL.md:** 持續更新，只保留最近學習

---

### Q4: 如何驗證記憶質量？

**A:** 質量檢查清單：
- [ ] 內容清晰易懂
- [ ] 格式統一
- [ ] 交叉引用正確
- [ ] 沒有重複內容
- [ ] 時間戳正確
- [ ] 文件大小合理（< 200 行）
