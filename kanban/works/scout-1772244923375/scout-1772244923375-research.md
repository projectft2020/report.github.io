## 研究摘要

本論文提出了一種基於共形預測（Conformal Prediction）的概念漂移定位新方法。傳統的漂移定位技術主要依賴局部統計檢驗，在高維度、低信號數據（如圖像流）中表現不佳。該研究將共形預測應用於基於模型的漂移定位思想中，提出了一種全局檢驗方案，能夠使用更廣泛的模型池，同時提供形式化的保證。實驗結果顯示，該方法在 Fashion-MNIST、NINCO 和新提出的 Fish-Head 數據集上均優於現有方法。

---

## 核心發現

1. **局部檢驗的根本缺陷** — 現有漂移定位方法面臨「三難取捨」問題：不使用或少用時間信息導致分組次優，改進分組則導致每組樣本不足，兩者都會降低檢驗效力。 | 來源: arXiv:2602.19790

2. **共形預測的全局檢驗方案** — 使用共形 p 值最小值作為漂移統計量，通過自舉法的 out-of-bag 樣本作為校準集，實現全局檢驗，避免局部分組問題。 | 來源: arXiv:2602.19790, Section 3

3. **顯著的性能優勢** — 在標準圖像數據集上，提出的方法（特別是使用 MLP 時）超越了現有技術；在更挑戰的 Fish-Head 數據集上，使用決策樹的版本表現最佳。 | 來源: arXiv:2602.19790, Section 4

## 詳細分析

### 1. 問題背景與動機

#### 1.1 概念漂移定義
在機器學習的流式學習和系統監控場景中，數據通常假設來自單一分佈。然而在實際應用中，分佈會隨時間變化，這種現象稱為**概念漂移**（Concept Drift）。

形式上，考慮無限觀察序列 X₁, X₂, ...，每個樣本 Xi 從潛在不同的分佈 Xi ~ Di 中抽取。當 Di ≠ Dj 時，就發生了概念漂移。

#### 1.2 漂移定位的定義
**漂移定位**（Drift Localization）的目標是找出所有受漂移影響的樣本。根據前人工作，這可以被形式化為局部和全局時間分佈不同的樣本集合：

L = { x : P(T|X=x) ≠ P(T) }

其中 T 是時間變量。換言之，如果給定樣本 x 時的條件時間分佈與全局時間分佈不同，則該樣本受漂移影響。

#### 1.3 現有方法的問題
現有的漂移定位方法主要基於**局部統計檢驗**：
- **無監督分組方法**：kdq-tree、quad-tree、k-means 等——不考慮時間信息
- **基於鄰域的方法**：如 LDD-DIS
- **基於模型的方法**：使用決策樹預測時間 T，但存在過擬合問題

**三難取捨問題**：
1. 不使用時間信息 → 分組次優 → 估計不準確
2. 使用更多信息 → 每組樣本少 → 檢驗效力低
3. 兩者都導致整體檢驗效力低

### 2. 共形預測基礎

#### 2.1 共形預測的概念
傳統分類器優化的是預期錯誤率，不提供單個預測正確性的保證。概率分類提供類別概率和不确定性度量，但模型可能過擬合或欠擬合。

**共形預測**提供替代方案，返回潛在類別的集合 F(x) ⊂ C，確保：
```
P[Y ∈ F(X)] ≥ α
```
對於任意高的置信度 α。通常最小化預測類別的期望數量。

#### 2.2 構造方法
最常見的方法是將類別級評分函數（如概率分類器）包裝到校準模型中。校準模型需要訓練在未使用的數據上，但因只處理簡單的一維評分，所需校準集可以較小。

通過共形 p 值，可以在訓練校準後選擇 α，構造 p_y(x) 使得對所有 α 都有：
```
P[Y ∉ {y : p_y(X) > α}] > 1 - α
```

### 3. 提出的方法

#### 3.1 核心思想
根據前人工作，一個點 x 不漂移當且僅當條件熵 H(T|X=x) 最大（假設有限時間域和均勻時間分佈），即對觀察時間點最大程度不確定。

使用共形預測的思路：
- 如果 c ∉ F(x)，則可以確定正確類別不是 c
- 如果能排除某個時間點，則拒絕 x 不漂移的假設
- 因此，當 F(x) ≠ T（時間點完整集合）時，拒絕 H₀

#### 3.2 統計量構造
使用共形 p 值表示 F_α：
```
F_α(x) = { y ∈ C : p_y(x) ≥ α }
```

拒絕 H₀ 如果最小共形評分小於 α，得到漂移 p 值：
```
p_drifting(X) = min_y p_y(X)
```

使用概率分類器作為類別級評分函數時，這對應於類別概率與全局概率的大偏差，即使樣本不漂移。

#### 3.3 算法實現

**算法 1：用於漂移定位的共形預測**

```
輸入：(x_i, y_i)_{i=1}^n, y_i ∈ C; n_boot ∈ N
輸出：(p_i)_{i=1}^n

1. P_i ← [] for all i
2. for t = 1, ..., n_boot do
3.     (I_in, I_oob) ← SampleBootstrap({1, ..., n})
4.     θ ← TrainModel(X_{I_in}, y_{I_in})
5.     for i ∈ I_in do
6.         P_i ← P_i + min_{c∈C} [ (1 + Σ_{k∈I_oob: y_k=c} 1[f(c|x_k,θ) ≤ f(c|x_i,θ)]) / (1 + |{k∈I_oob: y_k=c}|) ]
7.     end for
8. end for
9. p_i ← median(P_i) for all i
10. Return (p_i)_{i=1}^n
```

**關鍵設計選擇**：
- **自舉法**：out-of-bag 樣本構成自然的校準集
- **中位數聚合**：相當於檢驗的集成，當多數自舉導致拒絕時拒絕 H₀

#### 3.4 優勢分析

1. **校準集更小**：因只需校準一維信號，可使用更小校準集，允許更多樣本用於模型訓練
2. **可在訓練集上評估**：可在訓練模型的數據上評估樣本是否漂移，而非校準集
3. **模型無限制**：不要求模型誘導分組，兼容任何評分函數
4. **更大模型池**：可使用更廣泛的潛在模型，不僅限於能誘導分組的模型

### 4. 實驗評估

#### 4.1 數據集

**Fashion-MNIST**：
- 直接使用，無預處理
- 2×60 樣本，10 個漂移樣本

**NINCO (No ImageNet Class Objects)**：
- 使用深度嵌入 (DINOv2 ViT-S/14)
- 2×60 樣本，10 個漂移樣本

**Fish-Head（新提出）**：
- 基於 ImageNet 子集 "ImageNette"（10 個類別）
- 手動將 "tench" 類別分為兩個子類：魚頭朝左/朝右（漂移樣本）
- 其他樣本為非漂移樣本
- 使用 DINOv2 ViT-S/14 嵌入
- 2×250 樣本，98 個漂移樣本
- 在嵌入空間中，非漂移、漂移前、漂移後線性可分

#### 4.2 實驗設置

比較方法：
- **CP (本文)**：共形預測方法，使用決策樹 (DT) 和 MLP
- **MB-DL**：基於模型的漂移定位，使用排列和決策樹自舉
- **RF, heur.**：基於隨機森林的啟發式方法
- **LDD-DIS**：基於鄰域的方法
- **kdq-tree**：空間分割方法

評估指標：ROC-AUC，每個實驗重複 500 次

#### 4.3 自舉數量影響

觀察結果：
- 少量自舉時，顯著增加性能
- 大量自舉時，趨於平穩
- MLP 在約 75 個自舉時達到漸近狀態
- DT 在約 1,500 個自舉時穩定在 ROC-AUC ≈ 0.83
- 使用優化自舉方案（100 個）可獲得相當或更好的結果

樣本賦值次數約線性增長，速率 ≈ 0.64。超過 10 個自舉後，（幾乎）所有點都被賦值。

#### 4.4 方法性能比較

**Fashion-MNIST 和 NINCO**：
- 提出的共形預測定位方法優於相關工作
- 使用 MLP 時表現最佳

**Fish-Head**：
- 使用決策樹的共形預測方法優於其他方法
- 基於模型的方法與 MLP 版本表現相當
- 新數據基準更具挑戰性，需要更大樣本量確保優於隨機機會

### 5. 貢獻與創新點

1. **方法創新**：首次將共形預測應用於漂移定位，提供全局檢驗方案
2. **理論保證**：提供形式化的漂移檢測保證
3. **實用優勢**：擴展可用的模型池，減少對校準集大小的依賴
4. **新基準**：提出 Fish-Head 數據集，測試更細緻的漂移場景
5. **實驗驗證**：在多個數據集上顯著優於現有方法

### 6. 應用價值

#### 6.1 高維數據場景
該方法特別適合：
- **圖像流監控**：實時檢測圖像數據流中的漂移
- **計算機視覺系統**：監控視覺模型的輸入分佈變化
- **深度學習應用**：使用預訓練嵌入的高維特徵

#### 6.2 系統監控
- **工業檢測**：識別生產線數據中的異常漂移
- **金融風控**：檢測交易模式變化
- **網絡安全**：識別流量模式的漂移

#### 6.3 可解釋性
提供形式化的漂移解釋，結合 XAI 技術可以：
- 解釋哪些樣本受漂移影響
- 識別漂移的特徵模式
- 支持模型調整和再訓練決策

### 7. 局限性

1. **小樣本場景**：
   - Fish-Head 數據集需要更大樣本量（250×2）才能獲得優於隨機的結果
   - 少量漂移樣本時性能下降

2. **計算開銷**：
   - 需要多個自舉（建議 100+）
   - 每個自舉需要訓練獨立模型
   - 對於大型模型，計算成本可觀

3. **嵌入質量依賴**：
   - 方法性能依賴於使用的深度嵌入質量
   - 對於 NINCO，使用 DINOv2 嵌入
   - 不同嵌入可能導致性能差異

4. **二分類假設**：
   - 假設明確的「漂移前」和「漂移後」時間段
   - 複雜的漸進漂移場景可能需要擴展

5. **實現複雜性**：
   - 需要正確實現共形預測和自舉聚合
   - 對於新手用戶，門檻較高

### 8. 未來研究方向

作者提出的未來工作方向：
1. **小樣本優化**：開發更適合小樣本設置的方法，特別是少數漂移樣本的情況
2. **嵌入研究**：調查使用的深度嵌入的相關性和影響
3. **漸進漂移**：擴展到更複雜的漂移模式，如漸進漂移
4. **效率優化**：減少自舉數量或使用更高效的聚合策略
5. **理論分析**：進一步理論分析方法的覆蓋率和保證

### 9. 實現建議

根據實驗結果：
- **自舉數量**：建議 75-100 個，在性能和效率之間平衡
- **模型選擇**：
  - 圖像數據（嵌入空間）：推薦 MLP
  - 更複雜的漂移模式（如 Fish-Head）：推薦決策樹
- **校準集大小**：可使用相對較小的校準集（相比傳統方法）
