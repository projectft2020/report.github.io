# 對話式教學：任務優化系統

## 🎯 教學目標

讓 Charlie 孨會：
1. 分析任務複雜度
2. 選擇正確的模型（GLM-4.5 vs GLM-4.7）
3. 判斷何時需要分解任務
4. 使用時間預估和追蹤
5. 處理超時任務

---

## 📖 教學流程（7 個步驟）

### 第 1 步：測試現有能力

**你說**：
```
我想測試你對任務優化的了解。

給我分析這個任務的複雜度：
標題：建立專案索引地圖
輸入文件：0 個
代理類型：automation
請告訴我：
1. 複雜度等級（1-4）
2. 預估需要多少時間
3. 應該用什麼模型
```

**預期回應**：
```
📊 任務分析

任務：建立專案索引地圖

複雜度評估：
- 輸入文件：0 個
- 任務類型：automation
- 複雜度等級：1（簡單任務）

預估時間：3-8 分鐘
推薦模型：GLM-4.5

這是一個簡單任務，不需要分解，可以直接執行。
```

**如果正確**：
```
✅ 太好了！你已經理解基本概念。
繼續第 2 步 →
```

**如果需要提示**：
```
很接近了！讓我幫你理解：

複雜度等級判斷：
- 等級 1：簡單任務（0-1 個輸入，automation/creative）
- 等級 2：中等任務（1-2 個輸入）
- 等級 3：複雜任務（2-4 個輸入，analyst）
- 等級 4：極複雜任務（4+ 個輸入）

預估時間：
- 等級 1：3-8 分鐘
- 等級 2：8-18 分鐘
- 等級 3：15-35 分鐘
- 等級 4：30-50 分鐘

模型選擇：
- 等級 1-2：GLM-4.5（速度快）
- 等級 3-4：GLM-4.7（能力強）

再試一次分析這個任務？
```

### 第 2 步：教模型選擇規則

**你說**：
```
很好！現在讓我教你模型選擇的詳細規則。

GLM 模型有兩種主要選擇：
- GLM-4.5：速度快，但能力較弱
- GLM-4.7：速度較慢，但能力很強

選擇規則：
1. 複雜度等級 1 或 2 → 使用 GLM-4.5
2. 複雜度等級 3 或 4 → 使用 GLM-4.7
3. analyst 代理的任務 → 通常用 GLM-4.7
4. automation 簡單任務 → 可以用 GLM-4.5

現在告訴我：
這個任務該用什麼模型？
標題：策略回測驗證
輸入：1 個策略文件
代理：analyst
```

**預期回應**：
```
📊 任務分析

任務：策略回測驗證
輸入文件：1 個
任務類型：analyst

複雜度評估：
- analyst 代理 → 複雜度較高（+2）
- 1 個輸入文件（+1）
- 總分：3

複雜度等級：3（複雜任務）
預估時間：15-35 分鐘
推薦模型：GLM-4.7

原因：analyst 代理需要強大的分析能力，GLM-4.7 更適合。
```

**驗證要點**：
- 正確識別 analyst → 使用 GLM-4.7
- 給出合理的複雜度評估

### 第 3 步：教何時需要分解

**你說**：
```
很好！現在教你何時需要分解任務。

分解規則：
當任務「太複雜」時，我們應該把它拆成小任務。

需要分解的情況：
1. 輸入文件 ≥ 3 個
2. 複雜度等級 = 4
3. 複雜度等級 = 3 且是 analyst 任務
4. 複雜度分數 > 7

判斷以下任務是否需要分解：
任務 A：
- 輸入：4 個文件
- 代理：automation
- 複雜度：等級 3

任務 B：
- 輸入：1 個文件
- 代理：analyst
- 複雜度：等級 3
```

**預期回應**：
```
📋 分解建議分析

任務 A：
- 輸入文件：4 個 ✅ 超過閾值
- 需要分解：是
- 建議：按輸入文件分為 2 組
  - 子任務 1：處理文件 1-2
  - 子任務 2：處理文件 3-4

任務 B：
- 輸入文件：1 個（未超過閾值）
- 代理：analyst（複雜但可處理）
- 需要分解：否
- 建議：直接用 GLM-4.7 執行
```

**驗證要點**：
- 正確判斷任務 A 需要分解（≥ 3 個輸入）
- 正確判斷任務 B 不需要分解（只有 1 個輸入）
- 給出具體的分解建議

### 第 4 步：教分解策略

**你說**：
```
完美！現在教你如何分解任務。

三種分解策略：

1. 按輸入文件分解（最常見）
   - 適用：輸入文件 ≥ 3 個
   - 方法：每 1-2 個文件一組
   - 範例：4 個文件 → 2 個子任務

2. 按階段分解
   - 適用：有明確階段（分析→實作→驗證）
   - 方法：每個階段一個子任務

3. 按組件分解
   - 適用：系統集成類
   - 方法：設計→實作→測試

現在任務：
標題：風險評估系統
輸入：4 個策略文件
代理：analyst

請告訴我：
1. 是否需要分解？
2. 用什麼策略？
3. 如何分解？
```

**預期回應**：
```
📊 任務分解建議

任務：風險評估系統
輸入：4 個策略文件
代理：analyst

1. 是否需要分解？
✅ 是（輸入文件 ≥ 3）

2. 用什麼策略？
按輸入文件分解

3. 如何分解？
子任務 A：處理策略 1-2
子任務 B：處理策略 3-4

兩個子任務都使用 GLM-4.7，按順序執行。
```

### 第 5 步：教時間追蹤

**你說**：
```
很好！現在教你追蹤任務時間。

時間追蹤記錄：
- started_at：任務開始時間
- completed_at：任務完成時間
- actual_time_minutes：實際執行時間
- time_variance_percent：與預估的偏差

範例：
預估：15-35 分（中點 25 分）
實際：20 分
偏差 = (20-25)/25 × 100% = -20%（低於預估）

實際：40 分
偏差 = (40-25)/25 × 100% = +60%（超過預估）

請計算：
預估：8-18 分（中點 13 分）
實際：10 分
偏差 = ?
```

**預期回應**：
```
時間偏差計算：

預估中點：(8 + 18) / 2 = 13 分
實際時間：10 分
偏差 = (10 - 13) / 13 × 100% = -23.1%

結果：低於預估 23.1%
意義：任務執行效率高，比預估快
```

**驗證要點**：
- 正確計算預估中點
- 正確計算偏差百分比
- 理解偏差的意義

## 💡 教學技巧

### 1. 漸進式

從簡單開始，逐步增加複雜度：
- 步驟 1-3：基礎規則
- 步驟 4-5：實踐應用
- 步驟 6-7：綜合測試

### 2. 即時反饋

每個步驟立即驗證：
- 做對時：給予肯定
- 做錯時：具體指出問題
- 提供正確答案和說明

### 3. 總結規律

最後提供清晰的規則總結：
- 4 條核心規則
- 簡潔易記
- 便於應用
