## ❌ 常見錯誤與解決方案

### 錯誤 1：找不到 docker-compose.dev.yml

**症狀：**
zsh:1: command not found: docker-compose.dev

- 在錯誤目錄執行命令
- 缺少 `-f` 參數指定檔案

cd /Users/charlie/Dashboard  # 先 cd 到正確目錄
docker compose -f docker-compose.dev.yml ps


### 錯誤 2：不知道後端在哪個容器

- `curl http://localhost:8000` 返回前端 HTML
- `curl http://localhost:8001` 被攔截

- 混淆了生產環境和開發環境
- 不知道 API 實際運行在哪個容器

# 檢查容器狀態
docker ps --filter "name=dashboard"

# 開發環境 API：8001 端口
docker compose -f docker-compose.dev.yml exec backend curl http://localhost:8000/health

# 生產環境 API：8000 端口
docker exec dashboard_app curl http://localhost:8000/health


### 錯誤 3：不知道 API 路徑

- 405 Method Not Allowed
- 404 Not Found

- 不知道正確的路由路徑
- 沒有查看 API 文檔

1. 查看 Swagger UI：`http://localhost:8001/docs` 或 `http://localhost:8000/docs`
2. 查看 `main.py` 中的路由定義
3. 查看 `router` 的 `prefix` 配置


### 錯誤 4：回測結果沒有保存

- API 調用成功但沒有輸出
- 沒有找到輸出文件

- 忘記指定 `output_path`
- Agent 沒有寫入文件

- 總是在任務註釋中明確說明輸出路徑
- 任務完成後立即檢查該文件是否存在

### SKTASR 公式

SKTASR = Sharpe / [(1 - Skew/6) * (1 + (Kurt - 3)/6) * (1 + max(0, (3 - α)/3))]

其中：
- Sharpe: 夏普比率
- Skew: 偏度
- Kurt: 峰度
- α: 肥尾指數

複雜度分數 = 輸入文件數 + 任務類型係數 + 模型係數 + (依賴數 × 0.5) + (關鍵詞數 × 0.5)
