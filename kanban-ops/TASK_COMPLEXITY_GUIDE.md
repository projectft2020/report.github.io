# 任務複雜度估算使用指南

## 概述

`task_complexity.py` 提供自動化的任務複雜度估算功能，幫助在創建任務前：
- 評估任務複雜度等級（1-4 級）
- 預估執行時間
- 推薦最適合的模型
- 判斷是否需要分解任務

---

## 快速開始

### 1. 基本使用

```python
from task_complexity import TaskComplexityEstimator

# 創建估算器
estimator = TaskComplexityEstimator()

# 定義任務
task = {
    'input_paths': ['m001'],
    'agent': 'analyst',
    'model': 'zai/glm-4.7',
    'depends_on': [],
    'notes': '風險評估：VaR/CVaR 計算、極端場景分析'
}

# 估算複雜度
assessment = estimator.estimate(task)

# 查看結果
print(estimator.format_assessment(assessment))
```

**輸出**：
```
📊 複雜度評估

等級：3 - 複雜任務
分數：5.0
預估時間：15-35 分鐘
推薦模型：zai/glm-4.7

評估依據：
  輸入文件：1 個 (+1) | 任務類型：分析 (+2.0) | 模型：glm-4.7 (+1.5) | 關鍵詞：分析 (+0.5)
```

---

## 複雜度等級定義

| 等級 | 名稱 | 描述 | 預估時間 | 推薦模型 |
|------|------|------|----------|----------|
| **1** | 簡單任務 | 0-1 個輸入，輸出 <10KB | 3-8 分鐘 | GLM-4.5 |
| **2** | 中等任務 | 1-2 個輸入，輸出 10-30KB | 8-18 分鐘 | GLM-4.5 |
| **3** | 複雜任務 | 2-4 個輸入，輸出 30-60KB | 15-35 分鐘 | GLM-4.7 |
| **4** | 極複雜任務 | 4+ 個輸入，輸出 60-100KB | 30-50 分鐘 | GLM-4.7 + 分解 |

---

## 評估要素

複雜度分數 = (輸入文件數 × 1)
           + (任務類型係數)
           + (模型係數)
           + (依賴數 × 0.5)
           + (關鍵詞計數 × 0.5)

### 1. 輸入文件數量
- 每個輸入文件 +1 分

### 2. 任務類型係數
| 類型 | 係數 | 說明 |
|------|------|------|
| research | 1.0 | 研究任務 |
| analyst | 2.0 | 分析任務（最複雜）|
| automation | 1.0 | 自動化任務 |
| creative | 1.5 | 創作任務 |

### 3. 模型係數
| 模型 | 係數 | 說明 |
|------|------|------|
| GLM-4.7 | 1.5 | 處理能力強但較慢 |
| GLM-4.5 | 0.5 | 速度快但能力較弱 |
| GLM-4.5-flash | 0.3 | 最快但能力最弱 |

### 4. 依賴關係
- 每個依賴任務 +0.5 分

### 5. 關鍵詞計數
複雜關鍵詞（每個 +0.5 分）：
- 整合、集成、分析、對比、驗證
- 回測、優化、設計、實現、系統
- 架構、框架、多模組、跨系統
- 端對端、實證、模擬

---

## 複雜度分數對應

| 分數範圍 | 複雜度等級 |
|----------|-----------|
| ≤ 2 | 等級 1（簡單）|
| 3-4 | 等級 2（中等）|
| 5-7 | 等級 3（複雜）|
| > 7 | 等級 4（極複雜）|

---

## 使用建議

### 1. 模型選擇

```
等級 1-2 → 使用 GLM-4.5（速度快）
等級 3   → 使用 GLM-4.7（能力強）
等級 4   → 使用 GLM-4.7 + 考慮分解
```

### 2. 任務分解

**建議分解的情況**：
- 複雜度等級 4
- 複雜度等級 3 且輸入文件 ≥ 3 個

**分解策略**：
- 將輸入文件分組（每組 1-2 個）
- 創建 2-3 個子任務
- 每個子任務的複雜度 ≤ 等級 2

### 3. 時間預期

根據複雜度等級預估的時間範圍：
- **最小值**：樂觀情況（一切順利）
- **最大值**：保守情況（遇到問題）

如果實際執行時間超過最大值的 150%，應該：
1. 檢查是否出錯
2. 考慮重新評估複雜度
3. 下次分解任務

## 集成到工作流程

### 在創建任務前

```python
# 1. 先評估複雜度
task = {
    'input_paths': [...],
    'agent': 'analyst',
    ...
}
assessment = estimator.estimate(task)

# 2. 檢查是否需要分解
if assessment.should_decompose:
    print("⚠️ 建議分解任務")
    # 創建子任務...

# 3. 根據推薦選擇模型
task['model'] = assessment.recommended_model

# 4. 記錄預估時間
task['estimated_time'] = assessment.estimated_time

# 5. 創建任務
create_task(task)
```

### 在任務完成後

```python
# 1. 記錄實際時間
actual_time = task.end_time - task.start_time

# 2. 比較預估 vs 實際
estimated_min, estimated_max = task.estimated_time

if actual_time > estimated_max * 1.5:
    print("⚠️ 實際時間遠超預估，需要重新評估規則")
elif actual_time < estimated_min:
    print("✅ 執行效率高於預期")
else:
    print("✅ 在預估範圍內")
```
