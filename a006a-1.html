4. Code Patterns and Best Practices¶

4.1 Architecture Patterns¶

Layered Architecture¶

┌─────────────────────────────────────────┐
│         Presentation Layer              │  ← Frontend (React)
├─────────────────────────────────────────┤
│         API Gateway Layer               │  ← Middleware (Auth, CORS, Rate Limit)
├─────────────────────────────────────────┤
│         Router Layer                    │  ← FastAPI Routers
├─────────────────────────────────────────┤
│         Service Layer                   │  ← Business Logic
├─────────────────────────────────────────┤
│         Repository Layer               │  ← Data Access
├─────────────────────────────────────────┤
│         Storage Layer                   │  ← Databases, Cache, Files
└─────────────────────────────────────────┘

Key Principles: - Separation of Concerns: Each layer has a single responsibility - Dependency Direction: Upper layers depend on lower layers, never reverse - Interface-Based Programming: Use interfaces (abstract classes) for dependencies

Design Patterns Applied¶

Strategy Pattern:

# Abstract strategy interface
class Strategy(ABC):
    @abstractmethod
    def generate_signals(self, data: pd.DataFrame) -> pd.DataFrame:
        pass

# Concrete implementations
class RSIStrategy(Strategy):
    def generate_signals(self, data: pd.DataFrame) -> pd.DataFrame:
        # RSI-based signal generation
        pass

class MACDStrategy(Strategy):
    def generate_signals(self, data: pd.DataFrame) -> pd.DataFrame:
        # MACD-based signal generation
        pass

Factory Pattern:

class StrategyFactory:
    @staticmethod
    def create_strategy(strategy_type: str, **params) -> Strategy:
        if strategy_type == "rsi":
            return RSIStrategy(**params)
        elif strategy_type == "macd":
            return MACDStrategy(**params)
        else:
            raise ValueError(f"Unknown strategy type: {strategy_type}")

Repository Pattern:

class StrategyRepository(ABC):
    @abstractmethod
    def find_by_id(self, strategy_id: str) -> Optional[Strategy]:
        pass

    @abstractmethod
    def save(self, strategy: Strategy) -> Strategy:
        pass

    @abstractmethod
    def delete(self, strategy_id: str) -> bool:
        pass

Adapter Pattern:

class MarketDataProviderAdapter:
    def __init__(self, provider: str):
        self.provider = provider

    def fetch_data(self, symbol: str, start_date: date, end_date: date) -> pd.DataFrame:

from fastapi import HTTPException, status

class StrategyNotFound(HTTPException):

                "message": "Request validation failed",

@app.exception_handler(HTTPException)

4.4 Caching Pattern¶

Multi-Level Caching:

class CacheManager:
    def __init__(self):
        self.l1_cache = {}  # In-memory cache
        self.l2_cache = Redis()  # Redis cache

    async def get(self, key: str) -> Optional[Any]:
        # Try L1 first
        if key in self.l1_cache:
            return self.l1_cache[key]

        # Try L2
        value = await self.l2_cache.get(key)
        if value:
            self.l1_cache[key] = value  # Promote to L1
            return value

        return None

    async def set(self, key: str, value: Any, ttl: int = 300):
        # Set both L1 and L2
        self.l1_cache[key] = value
        await self.l2_cache.set(key, value, ex=ttl)

4.5 Async Task Pattern¶

Celery Task Definition:

from celery import Celery

celery_app = Celery('tasks', broker='redis://localhost:6379/0')

@celery_app.task(bind=True)
def run_backtest_task(self, strategy_id: str, config: dict):
    """Async backtest task"""
    try:
        # Execute backtest
        result = execute_backtest(strategy_id, config)

        # Save result
        save_backtest_result(result)

        # Update task status
        self.update_state(state='SUCCESS', meta={'result': result})

        return result
    except Exception as e:
        # Error handling
        handle_backtest_error(self.request.id, e)
        self.update_state(state='FAILURE', meta={'error': str(e)})
        raise

4.6 Logging Pattern¶

Structured Logging:

import logging
import json

class StructuredLogger:
    def __init__(self, name: str):
        self.logger = logging.getLogger(name)

    def log(self, level: str, message: str, **kwargs):
        log_entry = {
            "timestamp": datetime.utcnow().isoformat(),
            "level": level,
            "message": message,
            "context": kwargs
        }
        self.logger.info(json.dumps(log_entry))

Input Validation¶

from pydantic import BaseModel, Field, validator

class StrategyCreate(BaseModel):
    name: str = Field(..., min_length=1, max_length=100)
    type: StrategyType
    parameters: Dict

    @validator('name')
    def name_must_not_contain_special_chars(cls, v):
        if any(c in v for c in '<>"/\\|?*'):
            raise ValueError('名稱不能包含特殊字符')
        return v

10. Common Pitfalls and Solutions¶

10.1 Common Issues¶

Issue Impact Solution
Incomplete service layer High Complete service layer analysis, identify gaps
Inconsistent caching Medium Implement unified cache interface
Non-uniform error handling Medium Create unified exception classes, global handlers
Missing API version control Low Implement version migration strategy
Synchronous backtesting High Use Celery async tasks
Database connection pool bottlenecks Medium Increase pool size, monitor usage
Low cache hit rates Low Optimize caching strategy, increase TTL
Redis single point of failure High Use Redis Sentinel or Cluster

10.2 Anti-Patterns to Avoid¶

  1. Fat Controllers: Business logic should be in Service layer, not Router layer
  2. Direct Database Access in Router: Always go through Repository layer
  3. Hardcoded Configuration: Use environment variables or config files
  4. Silent Failures: Always log errors and return meaningful error messages
  5. N+1 Query Problem: Use eager loading or batch queries
  6. Leaking Sensitive Data: Never log passwords, tokens, or personal data
