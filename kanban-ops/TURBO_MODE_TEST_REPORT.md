## 修復的 Bug

### Bug 1: 浮點數錯誤

**錯誤：** `'float' object cannot be interpreted as an integer`

**原因：** `total_checks = duration_hours * 60 // check_interval_minutes` 在某些情況下產生浮點數

**修復：** 添加 `int()` 轉換
```python
total_checks = int(duration_hours * 60 // check_interval_minutes)
```

**測試：** ✅ 啟動成功，循環正常

### Bug 2: Scout 預熱失敗導致中斷

**錯誤：** Scout 腳本不存在時返回 `False`，導致執行中斷

**修復：** 改為返回 `True`，繼續執行
```python
if not scout_script.exists():
    self.log("INFO", "Scout 腳本不存在，跳過預熱")
    print(f"ℹ️  Scout 腳本不存在，跳過預熱")
    return True  # 返回 True 繼續執行
```

**測試：** ✅ Scout 預熱跳過，繼續執行

## 待改進項

### 優先級 1：wait_for_concurrent_tasks()

**問題：** 當前 `wait_for_concurrent_tasks()` 是空方法，沒有實際等待並行任務完成

**影響：**
- 測試時每次都觸發相同的任務（r004、f001）
- 無法追蹤並行任務狀態
- 無法控制並行數量

**建議實現：**
```python
def wait_for_concurrent_tasks(self, max_concurrent):
    """等待並行任務完成"""
    running_sessions = self.get_running_sessions()

    while len(running_sessions) >= max_concurrent:
        time.sleep(10)
        running_sessions = self.get_running_sessions()
```

### 優先級 2：任務狀態更新

**問題：** 沒有更新 Kanban 中的任務狀態

**影響：**
- 任務永遠是 `pending` 狀態
- 每次循環都觸發相同任務

**建議實現：**
```python
def update_task_status(self, task_id, status):
    """更新 Kanban 任務狀態"""
    with open(TASKS_JSON, 'r') as f:
        tasks = json.load(f)

    for task in tasks['tasks']:
        if task['id'] == task_id:
            task['status'] = status
            break

    with open(TASKS_JSON, 'w') as f:
        json.dump(tasks, f, indent=2)
```

創建 `auto_execute_queue.py`，讀取隊列並調用 `sessions_spawn`。
