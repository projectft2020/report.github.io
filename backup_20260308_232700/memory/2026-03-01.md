## 2026-03-01 11:24-11:36 - 分批建倉策略決策與研究進展

### 分批建倉決策（TW Supertrend 策略）

#### 用戶決策
**問題：** 感覺分批建倉比較能夠支撐經濟計畫的現金流

#### 分析與建議

**一次性建倉的問題：**
- 現金流衝擊：90,000 TWD → 現金瞬間歸零
- 風險暴露：進場後立即滿倉承擔全部風險
- 心理壓力：一次性大額建倉容易猶豫

**分批建倉的優勢：**
- ✅ 現金流壓力分散到多天
- ✅ 假突破時損失減少 50-70%
- ✅ 避免一次性追高
- ✅ 心理壓力更小，決策更理性
- ✅ 可根據趨勢強度動態調整

#### 推薦配置：時間分批 + 價格確認

**建倉規則：**
```
第 1 天（訊號出現）：33% 倉位
第 4 天：33% 倉位（確認趨勢仍有效）
第 7 天：33% 倉位（趨勢繼續確認）
總建倉期：6 天
```

**每次建倉確認條件：**
- 日線 Supertrend = 'long'
- 週線 Supertrend = 'long'
- 價格距離 Supertrend < 5%（避免追高）

**減倉規則：**
```
緊急平倉：任一 Supertrend 轉空 → 全部平倉
逐步減倉：價格跌破重要支撐位 → 減倉 1/3
```

**止損配置：**
```
任一分倉觸發止損 → 僅平掉該分倉
累計止損損失 > 總倉位 15% → 全部平倉
```

#### 建倉配置選項

**保守型（現金流緊張）：**
- 分批次數：4 次
- 時間間隔：5 天
- 每次比例：25%
- 總建倉期：15 天

**平衡型（推薦）：**
- 分批次數：3 次
- 時間間隔：3 天
- 每次比例：33%
- 總建倉期：6 天

**積極型（現金流充足）：**
- 分批次數：2 次
- 時間間隔：2 天
- 每次比例：50%
- 總建倉期：2 天

#### 實現要點

**核心原則：**
1. 結合日線+週線 Supertrend 雙重確認
2. 在每次建倉前確認趨勢仍然有效
3. 任一分倉觸發止損 → 對應分倉平倉
4. 累計止損損失 > 15% → 全部平倉

**回撤改善效果：**
- 假突破：損失減少 66%（9,000 → 3,000 TWD）
- 真突破：利潤略減但更安全（18,000 → 13,020 TWD）
- 現金流：分散到多天，支持經濟計畫

- 最穩定的體驗

- 接受更高波動率（~49% 回撤）

#### 實現技術

**1. 波動率估計（使用 EWMA）：**
```python
def estimate_volatility(returns, lambda_decay=0.94):
    # 指數加權移動平均
    weights = np.array([lambda_decay**i for i in range(len(returns))[::-1]])
    weights = weights / weights.sum()
    vol = np.sqrt(np.sum(weights * returns**2))
    return vol
```

**2. 波動率目標化倉位：**
```python
def volatility_targeting(current_vol, target_vol=0.15):
    current_vol_annual = current_vol * np.sqrt(252)
    target_vol_annual = target_vol
    scale_factor = target_vol_annual / current_vol_annual
    return min(max(scale_factor, 0.5), 2.0)  # 限制在 0.5-2.0 倍
```

**3. 波動率均等（推薦）：**
```python
def volatility_parity_portfolio(signals, volatilities, capital):
    # 計算每個標的的風險貢獻
    risk_contributions = signals * volatilities
    normalized_risk = risk_contributions / risk_contributions.sum()
    positions = normalized_risk * capital / volatilities
    return positions
```

#### 1. DeMiguel (2009) - Optimal vs Naive Diversification: 1/N Portfolio Strategy

**完成時間：** 2026-03-01 18:52
**子代理：** research
**運行時間：** 6 分 49 秒
**Tokens:** 652.5k（in 648.1k / out 4.4k）
**來源：** DeMiguel, Garlappi, Uppal (2009), The Review of Financial Studies

##### 核心發現

**震驚結論：**
「沒有一個複雜模型能夠在樣本外持續戰勝簡單的 1/N 等權重策略」

**測試範圍：**
- 14 個投資組合模型（均值-方差、貝葉斯、最小方差等）
- 7 個實證數據集（美股、國際股票、債券）
- 嚴格樣本外測試（不是內在過擬合）

**關鍵數據：**
| 資產數量 | 需要的估計窗口才能戰勝 1/N |
|---------|--------------------------|
| 25 種資產 | 250 年 |
| 50 種資產 | 500 年 |

**結論：** 在現實中，理論上的「最優」幾乎不可能實現

##### 為什麼 1/N 這麼強？

**核心問題：估計誤差 >> 最優化收益**
```
模型績效 = 真實最優收益 - 估計誤差

對於複雜模型：
- 需要估計的參數很多（均值、共變異數矩陣...）
- 估計誤差巨大
- 最優化帶來的收益完全被誤差吃掉
```

**實例：均值-方差模型**
- 50 種資產 → 需要估計 50 個均值 + 1,225 個共變異數 = **1,275 個參數**
- 用 10 年數據（2,500 天）→ 每個參數只有 2 個數據點
- 估計誤差巨大
- 「最優」組合實際上是「錯誤的組合」

**1/N 策略的優勢：**
- 不需要估計任何參數
- 不會有估計誤差
- 天然多樣化（每個資產權重相等）
- 簡單易懂，容易執行

##### 實踐啟示

**✅ 簡單性勝過複雜性**
- 不要過度優化
- 估計誤差是真實敵人
- 有時 1/N 就夠了

**✅ 資產選擇比權重優化更重要**
- 挑選好的資產 → 1/N 分配
- 選錯資產 → 再優化也沒用

**⚠️ 這不意味著「任何資產都可以 1/N」**
- 前提：資產有合理的期望報酬
- 前提：資產之間相關性不要太高
- 避免：顯然糟糕的資產（長期負報酬）

##### 對 QQQ/GLD/UUP 策略的啟示

**為什麼 1/N 有效？**
1. 資產質量好（QQQ 科技成長、GLD 黃金避險、UUP 美元）
2. 相關性適中（不是高相關的資產）
3. 各自服務不同風險因子

**簡化方向：**
- 不要過度優化權重
- 等權分配（1/N）可能就是最好的
- 把精力放在資產選擇和風險管理

**核心教訓：**
> 「有時最簡單的方法就是最好的方法」

#### 2. Qian (2016) - Risk Parity Fundamentals

**完成時間：** 2026-03-01 18:52
**子代理：** research
**運行時間：** 6 分 49 秒
**Tokens:** 652.5k（in 648.1k / out 4.4k）
**來源：** Edward Qian (2016), "Risk Parity Fundamentals"

##### 核心發現

**作者背景：**
- Edward Qian, PanAgora Asset Management CIO
- 2005 年首次提出「risk parity」這個術語
- 理論與實踐兼備的量化金融先驅

##### 核心概念：風險平價不是「等權」

**傳統誤區：60/40 組合的假象**
```
傳統 60/40 組合：
- 資本分配：60% 股票 + 40% 債券
- 風險分配：90%+ 股票 + <10% 債券

問題：這不是真正的多樣化！
```

**為什麼？**
- 股票波動率 ~15-20%
- 債券波動率 ~5-8%
- 風險貢獻 = 波動率 × 權重
- 股票的風險貢獻遠超債券

**風險平價的定義：**
```
傳統多樣化：基於資本分配
60% 股票 + 40% 債券

風險平價：基於風險貢獻
Risk Contribution_股票 = Risk Contribution_債券
```

##### 四大經濟制度：真正的多樣化

**Qian 的框架（基於 Bridgewater）：**
| 經濟制度 | 股票表現 | 債券表現 | 商品表現 | 推薦配置 |
|---------|---------|---------|---------|---------|
| 高增長 + 低通脹 | ✅ 表現好 | ❌ 表現差 | ❌ 表現差 | 高配股票 |
| 高增長 + 高通脹 | ⚠️ 風險高 | ❌ 表現差 | ✅ 表現好 | 商品+通脹保值 |
| 低增長 + 低通脹 | ❌ 表現差 | ✅ 表現好 | ❌ 表現差 | 高配債券 |
| 低增長 + 高通脹 | ❌ 表現差 | ❌ 表現差 | ⚠️ 風險高 | 困難時期 |

**風險平價的目標：**
- 在每種經濟制度下都有資產表現良好
- 不依賴單一制度
- 通過槓桿調整回報，而非資產選擇

##### 對 QQQ/GLD/UUP 的啟示

**風險平價版：**
```
步驟：
1. 計算風險貢獻
   - RC_QQQ = σ_QQQ × w_QQQ
   - RC_GLD = σ_GLD × w_GLD
   - RC_UUP = σ_UUP × w_UUP

2. 目標：RC_QQQ = RC_GLD = RC_UUP
   - 解方程得到權重

3. 調整槓桿
   - 目標：達到整體組合目標波動率（如 10%）
```

**對比 1/N：**
| 特性 | 1/N | 風險平價 |
|-----|-----|---------|
| 簡單性 | ✅ 極簡 | ⚠️ 中等 |
| 風險平衡 | ❌ 不平衡 | ✅ 完全平衡 |
| 槓桿使用 | ❌ 無槓桿 | ✅ 使用槓桿 |
| 估計誤差 | ✅ 無 | ⚠️ 需要估計波動率/相關性 |
| 經濟制度適應 | ⚠️ 中等 | ✅ 優秀 |

**建議：**
- 簡單開始 → 1/N 基準
- 風險敏感 → 風險平價增強
- 結合兩者 → 1/N 基準 + 風險平價調整

##### 關鍵機制

**波動率目標化（VT）的原理：**
```
低波動期 → 增加槓桿 → 維持目標風險暴露
高波動期 → 減少槓桿 → 控制風險
```

**為什麼有效？**
1. **槓桿效應**：報酬方向與波動率幅度負相關
   - 價格下跌時，波動率通常上升
   - 價格上漲時，波動率通常下降
   - 波動率調整無形中引入短期動量因子

2. **波動率集群**：金融市場的波動率呈現集群現象
   - 高波動期持續
   - 低波動期持續
   - 預測性較高

##### 實證結果（1980-2025）

```
VT 累積報酬：約 16,828%
VP 累積報酬：約 30,014%
VP+P 累積報酬：約 556,106% ← 樂透彩式分佈
```

**可可能貨期貨 2024 年案例：**
- VP+P 策略：單一交易貢獻超過 50% 投資組合報酬
- 比波動率目標化方法高近 10 倍

##### 實踐建議

**選擇適當的波動率目標：**
- 根據投資者風險承受能力和投資目標設定
- 通常建議目標波動率在 6-15% 之間
- 定期重新評估目標的適當性

**再平衡頻率設定：**
- 每週重新平衡是常見做法
- 可根據波動率變化速度調整頻率
- 考慮交易成本與反應速度的平衡

**風險限額管理：**
- 設定最大槓桿限制（通常為 2 倍）
- 建立壓力測試程序
- 監控相關性變化對整體風險的影響

##### 為什麼 GNN 在金融時間序列有效？

```
傳統方法：
- 把每隻股票獨立處理

```
時間維度：
- LSTM/GRU 捕捉時間依賴

空間維度（圖結構）：

融合：

```

##### 核心技術突破

**1. 動態圖構建**
- 不再使用靜態相關性矩陣
- 根據市場狀態動態調整圖結構
- 在不同市場制度下表現更好

**2. 注意力機制**
```
自注意力：
- 動態權重：重要股票權重更高

- 平衡風險貢獻
- 四大經濟制度多樣化
- 風險平價是增強工具

**3. 波動率目標化（2025）- VT, VP, VP+P：**
- VT：穩定、平滑、低波動
- VP：平衡之選（推薦）
- VP+P：極高回報但極端波動

**4. GNN 時空圖神經網絡：**
- 時空特徵融合
- 動態圖構建
- 預測精度提升 15-69%
