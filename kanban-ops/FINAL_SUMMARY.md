## 💡 核心功能

### 1. 複雜度估算（第 1 階段）

**功能**：
- 5 維度評估（輸入文件、任務類型、模型、依賴、關鍵詞）
- 4 級複雜度分類（1-4 級）
- 時間預估（最小-最大範圍）
- 模型推薦（GLM-4.5 vs GLM-4.7）

**公式**：
```
複雜度分數 = 輸入文件數 + 任務類型係數 + 模型係數 +
             (依賴數 × 0.5) + (關鍵詞數 × 0.5)

分數 → 等級 → 預估時間 → 推薦模型
≤2  → 1   → 3-8分   → GLM-4.5
3-4  → 2   → 8-18分  → GLM-4.5
5-7  → 3   → 15-35分 → GLM-4.7
>7   → 4   → 30-50分 → GLM-4.7 + 分解
```

### 2. 時間追蹤（第 2 階段）

**功能**：
- 記錄任務開始/完成時間
- 計算實際執行時間
- 計算時間偏差百分比
- 生成統計報告
- 識別異常任務

**新增字段**：
```json
{
  "time_tracking": {
    "estimated_time": {"min": 3, "max": 8},
    "complexity_level": 1,
    "recommended_model": "zai/glm-4.5",
    "started_at": "2026-02-19T21:30:00",
    "completed_at": "2026-02-19T21:35:00",
    "actual_time_minutes": 5.2,
    "time_variance_percent": -12.5
  }
}
```

### 3. 自動分解建議（第 3 階段）

**功能**：
- 自動判斷是否需要分解
- 三種分解策略（by_input, by_phase, by_component）
- 生成具體的子任務方案
- 預估時間節省

**分解規則**：
```
需要分解 if:
  - 複雜度等級 4
  - 複雜度等級 3 且 輸入文件 ≥ 3
  - 複雜度等級 3 且 agent = analyst
  - 複雜度分數 > 7
```

**三種策略**：
1. **by_input**：按輸入文件分組
2. **by_phase**：按階段（分析→實作→驗證）
3. **by_component**：按組件（設計→實作→集成）

### 4. 超時處理（第 4 階段）

**功能**：
- 自動檢測超時任務（閾值 30 分鐘）
- 檢查輸出文件完整性
- 四種推薦動作
- 重試策略生成

**四種動作**：
| 動作 | 條件 | 說明 |
|------|------|------|
| mark_completed | 輸出存在且完整 | 超時但成功 |
| mark_failed | 輸出存在但不完整 | 執行中斷 |
| retry | 輸出不存在且超時 | 需要重試 |
| wait | 仍在預估範圍內 | 繼續等待 |

## 🔄 完整工作流程

### 創建任務前

```bash
# 1. 先分析複雜度
python3 task_manager.py analyze p006

# 2. 如果複雜度高，查看分解建議
python3 task_manager.py decompose p006

# 3. 決定直接執行或分解
```

### 執行任務時

```python
from time_tracker import TaskTimeTracker

tracker = TaskTimeTracker('/path/to/tasks.json')

# 任務開始
tracker.mark_task_started('p006')

# ... 執行 ...

# 任務完成
tracker.mark_task_completed('p006')
# 自動計算實際時間和偏差
```

### 定期檢查

```bash
# 1. 查看統計報告
python3 task_manager.py stats

# 2. 檢查超時任務
python3 task_manager.py timeout

# 3. 處理超時任務
# 根據建議執行 mark_completed, mark_failed, 或 retry
```

### 從失敗中學習

**h004 教訓**：
1. 複雜度等級 3 + 4 個輸入文件 → 必須分解
2. GLM-4.5 不足以處理複雜任務
3. 超時不意味著失敗，需要檢查輸出

**m002 成功經驗**：
1. GLM-4.7 可以處理複雜任務
2. 預估時間（15-35 分）涵蓋實際（28.4 分）
3. analyst 代理需要更長時間但可以完成

```
5. 返回步驟 1
```
