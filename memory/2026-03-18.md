## 🔧 修復方案

### 【選項 A】設置系統環境變量（推薦）⭐⭐⭐⭐⭐

**步驟：**

1. **打開 shell 配置文件**
   ```bash
   # 如果使用 zsh（macOS 預設）
   nano ~/.zshrc
   
   # 如果使用 bash
   nano ~/.bash_profile
   ```

2. **添加環境變量**
   ```bash
   # 在文件末尾添加：
   export TELEGRAM_BOT_TOKEN="your_actual_bot_token_here"
   ```

3. **重新加載配置**
   ```bash
   # zsh
   source ~/.zshrc
   
   # bash
   source ~/.bash_profile
   ```

4. **重啟 Gateway 服務**
   ```bash
   openclaw gateway stop
   openclaw gateway start
   ```

**優點：**
- ✅ 安全性最佳（Token 不在配置文件中）
- ✅ 符合最佳實踐
- ✅ 一次性設置，永久有效

---

**步驟：**

1. **編輯配置文件**

**適用場景：**

## 核心學習

### 環境變量 vs 配置文件

**最佳實踐：**
- **環境變量：** 適合敏感信息（API keys, tokens, passwords）
- **配置文件：** 適合非敏感信息（paths, urls, feature flags）

**安全性考量：**
- 環境變量：不會意外提交到 Git（除非明確寫入）
- 配置文件：需要明確加入 .gitignore，仍有風險

---

### OpenClaw Gateway 問題診斷流程

**診斷步驟：**
1. 查看日誌檔案（/tmp/openclaw/openclaw-YYYY-MM-DD.log）
2. 識別錯誤類型（404, connection refused, timeout）
3. 追蹤重試次數和模式
4. 檢查相關配置（openclaw.json）
5. 驗證環境變量設置
6. 提供多個修復方案（按優先級排序）

---

### Telegram Bot API 錯誤代碼

**404 Not Found：**
- Bot Token 無效或過期
- Bot Token 配置錯誤
- Bot 尚未配置（第一次設置）

**其他常見錯誤：**
- 401 Unauthorized：Token 格式錯誤
- 409 Conflict：Webhook 已存在
- 429 Too Many Requests：Rate limit
