# 模型选择规则

## 核心原则

**GLM-4.5（速度快）**
- 适用：复杂度等级 1-2
- 优势：响应快、成本低
- 劣势：能力较弱，不适合复杂任务

**GLM-4.7（能力强）**
- 适用：复杂度等级 3-4
- 优势：分析能力强、理解深入
- 劣势：响应较慢、成本较高

**GLM-4.7-Flash（仅限主协调）**
- 适用：主 Orchestrator 使用
- 限制：并发数 1（不可用于子代理）
- 用途：快速协调和简单响应

## 选择规则

### 规则 1：基于复杂度
```
等级 1-2 → GLM-4.5
等级 3-4 → GLM-4.7
```

### 规则 2：基于任务类型
```
automation 简单任务 → GLM-4.5
research → GLM-4.5（默认）
analyst → GLM-4.7（强制）
creative → GLM-4.7
```

### 规则 3：并发限制
```
GLM-4.7-Flash = 1 并发 → 永远不用于子代理
GLM-4.5 = 10 并发 → 适合并行任务
GLM-4.7 = 5 并发 → 适合质量任务
```

## 决策树

```
开始
  ↓
任务类型 = analyst？
  ├─ 是 → GLM-4.7
  └─ 否 → 计算复杂度分数
           ↓
         复杂度等级 ≥ 3？
           ├─ 是 → GLM-4.7
           └─ 否 → GLM-4.5
```

## 实际案例

### 案例 1：简单 automation 任务
- 任务：文件格式转换
- 复杂度：等级 1
- 选择：GLM-4.5 ✅

### 案例 2：中等 analyst 任务
- 任务：策略回测分析
- 复杂度：等级 3
- 选择：GLM-4.7 ✅

### 案例 3：复杂 analyst 任务（4 个输入）
- 任务：多策略对比
- 复杂度：等级 4
- 选择：GLM-4.7 + 分解 ✅

### 案例 4：research 任务
- 任务：Web 搜索和信息收集
- 复杂度：等级 2
- 选择：GLM-4.5（默认）✅

## 注意事项

1. **analyst 任务总是用 GLM-4.7**
   - 即使复杂度较低，analyst 也需要深度分析能力

2. **并行任务优先 GLM-4.5**
   - GLM-4.5 支持 10 并发，适合批量处理

3. **GLM-4.7-Flash 仅用于主协调**
   - 子代理会阻塞，永远不要在子代理中使用

4. **质量优于速度**
   - 复杂任务（等级 3-4）应该用 GLM-4.7，即使慢一些
