Detailed Analysis¶

Theoretical Background¶

Functional decomposition is a fundamental concept in machine learning interpretability that provides a theoretical foundation for decomposing complex prediction functions into interpretable components. At its core, functional decomposition represents a high-dimensional prediction function as a sum of individual feature effects and interaction effects:

[\hat{f}(\mathbf{x}) = \sum_{S\subseteq{1,\ldots,p}} \hat{f}_S(\mathbf{x}_S)]

where (S) represents subsets of features, (\hat{f}S) represents functional components, and (\mathbf{x}_S) is the vector of features in the index set (S). This decomposition includes: - Intercept term ((\hat{f}_0)): The baseline prediction - Main effects ((\hat{f}_j)): Individual feature contributions - Interaction effects ((\hat{f}, \ldots)): Joint contributions of feature groups}, \hat{f}_{i,j,k

The theoretical significance of functional decomposition lies in its connection to game theory, where features are treated as players in a cooperative game, and the prediction represents the total payout to be distributed fairly among the players.

Shapley Interaction Index¶

The Shapley interaction index extends traditional Shapley values to capture cooperative effects between features. While standard Shapley values (\phi_j) represent individual feature contributions, the Shapley interaction index (\phi_{i,j}) quantifies the joint contribution of features (i) and (j):

[\phi_{i,j} = \sum_{S\subseteq\backslash{i,j}} \frac{|S|!(M-|S|-2)!}{2(M-1)!} \delta_{ij}(S)]

where (\delta_{ij}(S) = \hat{f}{\mathbf{x}}(S\cup{i,j}) - \hat{f}}}(S\cup{i}) - \hat{f{\mathbf{x}}(S\cup{j}) + \hat{f}(S)).}

This formulation subtracts the main effects of the individual features, isolating the pure interaction effect. The resulting index satisfies important properties: - Symmetry: Features with identical interaction effects receive equal values - Linearity: The interaction index is linear in the prediction function - Dummy property: Features that contribute no interaction receive zero values - Efficiency: The sum of all interaction terms equals the total prediction minus the baseline

Decomposition Algorithms¶

1. Functional ANOVA¶

Functional ANOVA, proposed by Hooker (2004), provides a principled approach to decomposition under the assumption of independent features. The components are defined as:

[\hat{f}S(\mathbf{x}) = \int}} \left( \hat{f}(\mathbf{x}) - \sum_{V \subset S} \hat{fV(\mathbf{x}) \right) d \mathbb{P}(X)]

This method satisfies desirable axioms: - Zero means: All effects are centered around zero - Orthogonality: Components do not share information - Variance decomposition: Total variance equals the sum of component variances

2. Generalized Functional ANOVA¶

For dependent features, Hooker (2007) proposed generalized functional ANOVA, which relaxes the independence assumption. The components are defined through projection onto additive functions:

[\hat{f}S(\mathbf{x}_S) = \arg\min^S){S \in P}} \int \left(\hat{f}(\mathbf{x}) - \sum]} g_S(\mathbf{x}_S)\right)^2 w(\mathbf{x})\,d\mathbf{x

This method satisfies hierarchical orthogonality, ensuring that components for subsets are orthogonal to their supersets.

3. Accumulated Local Effects (ALE)¶

ALE plots provide a functional decomposition with pseudo-orthogonality properties. The ALE operator (H_S) maps functions to their ALE plots for feature subset (S). The key property is that applying different operators yields zero:

[H_S(H_U(\hat{f})) = 0 \text{ for } S \neq U]

This pseudo-orthogonality prevents entanglement of marginal effects and allows hierarchical computation.

4. TreeSHAP¶

For tree-based models, TreeSHAP leverages the tree structure to compute Shapley values efficiently. The complexity reduces from (O(TL2^M)) for exact methods to (O(TLD^2)), where: - (T) = number of trees - (L) = maximum number of leaves - (D) = maximum tree depth - (M) = number of features

TreeSHAP comes in two variants: - Interventional: Computes classic Shapley values - Tree-path dependent: Computes conditional Shapley values

Complexity Analysis¶

The computational complexity of different decomposition methods varies significantly:

Exact Methods¶

  • Functional ANOVA: (O(2^p)) - exponential in the number of features
  • Shapley values: (O(p!)) - factorial complexity due to permutation evaluation
  • Shapley interactions: (O(2^p \cdot p^2)) - exponential with polynomial factor

This exponential complexity makes exact methods infeasible for high-dimensional problems (p > 20).

Approximation Methods¶

  • Monte Carlo Shapley: (O(M \cdot p \cdot C)), where (M) is the number of samples and (C) is model evaluation cost
  • TreeSHAP: (O(TLD^2)) - efficient for tree ensembles
  • Permutation method: (O(M \cdot p \cdot C)) with better variance properties than Monte Carlo
  • KernelSHAP: (O(K \cdot p \cdot C)), where (K) is the number of coalition samples

Recent Optimizations¶

Recent advances have focused on: - Structural assumptions: Exploiting sparsity or low effective dimensionality - Approximation algorithms: Providing probabilistic guarantees with limited resources - Hardware acceleration: GPU implementations for massive parallel computation - Approximation-optimality tradeoffs: Balancing computational efficiency with theoretical guarantees

Practical Examples¶

1. Survival Model Interpretation¶

Recent work by Langbein et al. (2026) introduced Survival Functional Decomposition (SurvFD) for analyzing feature interactions in survival models. This approach decomposes higher-order effects into time-dependent and time-independent components:

[\hat{f}(t, \mathbf{x}) = \hat{f}0(t) + \sum} \hat{fj(t, x_j) + \sum(t, x_i, x_j) + \ldots]} \hat{f}_{i,j

The corresponding SurvSHAP-IQ extends Shapley interactions to time-indexed functions, providing insights into when and why additive explanations fail in survival analysis.

2. Computer Vision Applications¶

In vision models, Shapley interactions help explain how different image regions (superpixels) cooperate to influence predictions. The shapiq package enables explanation of vision transformers by computing any-order interactions between image patches.

3. Natural Language Processing¶

For language models, functional decomposition can reveal how different words or phrases interact to affect predictions. This is particularly valuable for understanding model behavior in tasks like sentiment analysis or text classification.

4. Tabular Data Analysis¶

In traditional machine learning applications with tabular data, Shapley interactions help identify: - Important feature pairs: Which combinations of features have the strongest joint effects - Interaction patterns: Whether interactions are synergistic (positive) or antagonistic (negative) - Model behavior: How different model architectures capture different types of interactions

Applications in Model Interpretation¶

Global Model Understanding¶

Functional decomposition provides a comprehensive framework for global model interpretation: - Feature importance: Aggregating absolute Shapley values across the dataset - Interaction importance: Measuring the strength of feature interactions globally - Model visualization: Creating dependence plots that show both main effects and interactions

Local Instance Explanation¶

For individual predictions, Shapley interactions offer: - Detailed attribution: Explaining how feature combinations contribute to specific predictions - Counterfactual insights: Understanding how changes in multiple features would affect predictions - Trust assessment: Evaluating whether model decisions are based on reasonable feature interactions

Model Debugging and Improvement¶

Functional decomposition helps in: - Identifying spurious correlations: Detecting when models rely on unrealistic feature interactions - Improving model design: Guiding the development of more interpretable model architectures - Fairness assessment: Evaluating whether models treat different demographic groups fairly
