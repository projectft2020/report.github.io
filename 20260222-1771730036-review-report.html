s101b 設計要求：

Data Source Error
├── Retry (3次，指數退避)
├── Fallback → 返回緩存數據

實際實現：

// DashboardService 中沒有錯誤處理
async generateDashboard(): Promise<DashboardData> {
  const [metrics, alerts, status] = await Promise.all([
    this.dataFetcher.fetchMetrics(),
    this.dataFetcher.fetchAlerts(),
    this.dataFetcher.fetchStatus()
  ]);
  // ... 如果任何一個拋出異常，整個流程會失敗
}

建議：

async generateDashboard(): Promise<DashboardData> {
  try {
    const [metrics, alerts, status] = await Promise.all([
      this.fetchWithRetry(() => this.dataFetcher.fetchMetrics()),
      this.fetchWithRetry(() => this.dataFetcher.fetchAlerts()),
      this.fetchWithRetry(() => this.dataFetcher.fetchStatus())
    ]);
    // ...
  } catch (error) {
    // Fallback: 返回緩存數據
    return this.getCachedDashboard() || this.getDegradedDashboard();
  }
}

private async fetchWithRetry<T>(fetchFn: () => Promise<T>): Promise<T> {
  let lastError: Error;
  for (let i = 0; i < 3; i++) {
    try {
      return await fetchFn();
    } catch (error) {
      lastError = error;
      if (i < 2) {
        await this.sleep(Math.pow(2, i) * 1000); // 指數退避
      }
    }
  }
  throw lastError;
}

問題 2.4：健康評分邏輯有誤 (中優先級)¶

當前實現：

private calculateHealthScore(metrics: Map<string, Metric>): number {
  let totalScore = 0;
  let weightCount = 0;

  metrics.forEach((metric) => {
    if (metric.threshold) {
      const score = Math.max(0, 100 - (metric.value / metric.threshold) * 100);
      totalScore += score;
      weightCount++;
    }
  });

  return weightCount > 0 ? Math.round(totalScore / weightCount) : 100;
}

問題分析：

假設 CPU = 85%, threshold = 80%： - 計算：100 - (85 / 80) * 100 = 100 - 106.25 = -6.25 - Math.max(0, -6.25) = 0

這意味著 CPU 超過閾值 5% 時，健康分數直接變為 0，這太嚴格了。

建議改進：

private calculateHealthScore(metrics: Map<string, Metric>): number {
  let totalScore = 0;
  let weightCount = 0;

  metrics.forEach((metric) => {
    if (metric.threshold) {
      // 健康分數 = 100 * (1 - 使用率/閾值)，但不低於 0
      const usageRatio = metric.value / metric.threshold;
      let score = 100 * (1 - usageRatio);

      // 超過閾值時，每超過 1% 扣 10 分
      if (usageRatio > 1) {
        score = 100 - (usageRatio - 1) * 1000;
      }

      score = Math.max(0, score);
      totalScore += score;
      weightCount++;
    }
  });

  return weightCount > 0 ? Math.round(totalScore / weightCount) : 100;
}

問題 2.5：XSS 安全風險 (中優先級)¶

位置： SimpleHTMLDashboard.render()

private renderMetrics(metrics: Map<string, Metric>): string {
  // ❌ 直接插入 metric.name，未轉義
  html += `<div class="metric"><h4>${metric.name} ${status}</h4>...</div>`;
}

雖然是模擬數據，但應養成安全習慣。建議：

private escapeHtml(unsafe: string): string {
  return unsafe
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&#039;");
}

private renderMetrics(metrics: Map<string, Metric>): string {
  let html = '';
  metrics.forEach((metric) => {
    const status = metric.threshold && metric.value > metric.threshold ? '⚠️' : '✅';
    html += `
      <div class="metric">
        <h4>${this.escapeHtml(metric.name)} ${status}</h4>
        <p><strong>${this.escapeHtml(metric.value + metric.unit)}</strong></p>
        ${metric.threshold ? `<p>Threshold: ${this.escapeHtml(metric.threshold + metric.unit)}</p>` : ''}
      </div>`;
  });
  return html;
}
