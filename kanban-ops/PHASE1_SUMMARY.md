## 📊 複雜度評估規則

### 評估公式

```
複雜度分數 = (輸入文件數 × 1)
           + (任務類型係數)
           + (模型係數)
           + (依賴數 × 0.5)
           + (關鍵詞計數 × 0.5)
```

### 等級對應

| 分數範圍 | 複雜度等級 | 預估時間 | 推薦模型 |
|----------|-----------|----------|----------|
| ≤ 2 | 等級 1（簡單）| 3-8 分 | GLM-4.5 |
| 3-4 | 等級 2（中等）| 8-18 分 | GLM-4.5 |
| 5-7 | 等級 3（複雜）| 15-35 分 | GLM-4.7 |
| > 7 | 等級 4（極複雜）| 30-50 分 | GLM-4.7 |

### 分解建議

```
if (等級 == 4) or (等級 == 3 and 輸入文件 >= 3):
    建議分解 = true
```

## 🎯 使用建議

### 模型選擇規則

| 複雜度 | 推薦模型 | 理由 |
|--------|----------|------|
| 等級 1-2 | GLM-4.5 | 簡單任務，速度快 |
| 等級 3 | GLM-4.7 | 複雜任務，需要更強推理 |
| 等級 4 | GLM-4.7 + 分解 | 極複雜，建議分解 |

### 任務分解策略

**何時分解**：
- 複雜度等級 4
- 複雜度等級 3 且輸入文件 ≥ 3 個

**如何分解**：
- 輸入文件分組（每組 1-2 個）
- 創建 2-3 個子任務
- 確保每個子任務複雜度 ≤ 等級 2
